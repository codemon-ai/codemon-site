## 4. 기술적 제한/리스크

> 출처: `known-limitations.md` + 문서 분석

| # | 제한 | 영향 | 우회/상태 |
|---|------|------|----------|
| 1 | **Supabase DATABASE\_URL 미수신** | DB 연결 테스트 불가 — **Wave 0 블로커** | 클라이언트에게 요청 필요 |
| 2 | Drizzle ORM v0.x 고정 | Better Auth와 1.0 비호환 | drizzle-orm@^0.38.0 사용 |
| 3 | Supabase Transaction Pooler | Prepared Statements 불가 | prepare: false 설정 |
| 4 | @react-pdf/renderer | 공유 패키지 배치 불가 (Turborepo 크래시) | apps/admin에만 설치 |
| 5 | Better Auth 비밀번호 해시 | v1 bcrypt ↔ Better Auth scrypt 비호환 | 하이브리드 verify 구현됨 |
| 6 | Next.js after() 실험적 | experimental.after 필요 | 설정 완료 |
| 7 | 포트 충돌 (3001, 3002) | v1이 점유 중 | v2는 3000, 3010 사용 |

> 🚨 **블로커**: Supabase DATABASE\_URL을 클라이언트로부터 받아야 v2 개발 시작 가능
