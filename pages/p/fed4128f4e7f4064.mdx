## 6. 현재 비즈니스 플로우 (코드 기준)

### 예약 플로우
```
고객 상품 선택 → 장바구니 → 결제(토스) → Booking(PENDING)
  → 관리자 인보이스 발행(INVOICE_SENT) → 결제확인(PAYMENT_COMPLETED)
  → 관리자 바우처 발행(VOUCHER_SENT) → 이용완료(COMPLETED)
```

### ⚠️ COO가 원하는 플로우
```
고객 상품 선택 → 장바구니 → Order 생성(Requested)
  → 관리자 검증(Pending_check) → 인보이스 발송(Payment_requested)
  → 계약금 또는 완납(Payment_received) → 벤더 예약 요청(Vendor_requested)
  → 벤더 확정(Confirmed) → 바우처 발송(Voucher_sent)
  → 서비스 제공(In_service) → 완료(Completed)
```

**핵심 차이점**:
1. Order가 Booking 위에 존재
2. 계약금/잔금 2단계 결제
3. 벤더 확정 단계가 결제 후에 별도 존재
4. In_service 상태 별도
