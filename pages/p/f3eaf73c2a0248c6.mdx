# Claude Code 개발 위임 가이드

> OpenClaw 봇에서 Claude Code에 개발 작업을 일임하고, 완료 시 자동 알림받는 워크플로우

---

## 1. 핵심 명령어

```bash
claude -p --dangerously-skip-permissions \
  --disallowedTools 'Bash(rm:*)' \
  --verbose --output-format stream-json \
  '작업 내용'
```

### 필수 플래그

| 플래그 | 역할 | 빠지면? |
|--------|------|---------|
| `-p` | **non-interactive 모드** | ❌ interactive로 떠서 입력 대기하며 멈춤 |
| `--dangerously-skip-permissions` | 모든 권한 자동 승인 (YOLO) | ❌ 매번 승인 물어봄 |
| `--disallowedTools "Bash(rm:*)"` | rm 명령어 차단 (안전장치) | ⚠️ 파일 삭제 위험 |
| `--verbose` | stream-json 사용 시 필수 | ❌ 에러남 |
| `--output-format stream-json` | JSON 스트림 출력 (진행 확인용) | 선택사항 |

## 2. OpenClaw에서 실행하기

### 기본 패턴

OpenClaw 봇의 `exec` 도구로 Claude Code를 백그라운드 실행하고, 끝나면 wake 알림을 보내는 구조:

```bash
exec pty:true workdir:{프로젝트경로} background:true command:"claude -p --dangerously-skip-permissions --disallowedTools 'Bash(rm:*)' --verbose --output-format stream-json '작업내용' && openclaw gateway wake --text 'Done: {요약}' --mode now"
```

**exec 옵션 설명:**

| 옵션 | 역할 |
|------|------|
| `pty:true` | 코딩 에이전트는 TTY 필요 |
| `background:true` | 백그라운드 실행 (봇이 다른 일 가능) |
| `workdir:{경로}` | 프로젝트 루트 디렉토리 |
| `&& openclaw gateway wake` | 작업 끝나면 봇에게 알림 |
| `--mode now` | 즉시 알림 (heartbeat 안 기다림) |

### ⚠️ 실패하는 패턴들

```bash
# ❌ -p 없음 → interactive 모드로 떠서 멈춤
claude --dangerously-skip-permissions "프롬프트"

# ❌ --verbose 없이 stream-json → 에러
claude -p --dangerously-skip-permissions --output-format stream-json "프롬프트"

# ❌ pty 없음 → TTY 관련 에러
exec background:true command:"claude -p ..."
```

## 3. 실전 예시

### 예시 1: 간단한 버그 수정

```bash
exec pty:true workdir:~/workspace/myproject background:true command:"claude -p --dangerously-skip-permissions --disallowedTools 'Bash(rm:*)' --verbose --output-format stream-json '로그인 시 세션 만료 안 되는 버그 수정해줘. 완료되면 git commit & push까지.' && openclaw gateway wake --text 'Done: 로그인 버그 수정 완료' --mode now"
```

### 예시 2: GitHub Issue 기반 작업

```bash
exec pty:true workdir:~/workspace/bluemango-plan background:true command:"claude -p --dangerously-skip-permissions --disallowedTools 'Bash(rm:*)' --verbose --output-format stream-json '
## 작업: GitHub Issue #60 해결

### 요구사항
- gh issue view 60 으로 이슈 내용 확인
- dev 브랜치에서 feature 브랜치 생성
- 이슈에 명시된 해결 방안대로 구현
- 테스트 확인
- git commit & push
- PR 생성 (base: dev)
' && openclaw gateway wake --text 'Done: Issue #60 해결 완료' --mode now"
```

### 예시 3: 여러 이슈 순차 처리

```bash
exec pty:true workdir:~/workspace/bluemango-plan background:true command:"claude -p --dangerously-skip-permissions --disallowedTools 'Bash(rm:*)' --verbose --output-format stream-json '
## 작업: GAP 이슈 일괄 해결

GitHub Issues #60~#67을 순서대로 해결해줘.

### 규칙
- dev 브랜치 기준
- 각 이슈별로 gh issue view {번호}로 내용 확인
- 코드 수정 후 테스트
- 이슈별 커밋 분리
- 완료된 이슈는 커밋 메시지에 closes #{번호} 포함
- 전부 끝나면 push
' && openclaw gateway wake --text 'Done: GAP 이슈 일괄 해결' --mode now"
```

## 4. 진행 상황 확인

```bash
# 실행 중인 세션 목록
process action:list

# 특정 세션 로그 확인 (마지막 50줄)
process action:log sessionId:{세션ID} limit:50

# 세션 상태 확인
process action:poll sessionId:{세션ID}

# 필요시 종료
process action:kill sessionId:{세션ID}
```

## 5. 안전 수칙

### 실행 전 체크리스트

```
✅ git init 되어 있는가?
✅ 현재 브랜치 확인 (dev? main?)
✅ uncommitted 변경사항 없는가?
✅ workdir 경로가 정확한가?
```

### 권장 안전장치

| 장치 | 방법 |
|------|------|
| **rm 차단** | `--disallowedTools "Bash(rm:*)"` |
| **체크포인트** | 작업 전 `git commit -m "checkpoint"` |
| **복구** | 망하면 `git reset --hard HEAD` |
| **도구 제한** | 읽기만 필요하면 `--tools "Read,Edit,Grep,Glob"` |

### YOLO 모드 옵션 비교

```bash
# 기본 YOLO (모든 권한)
claude -p --dangerously-skip-permissions "작업"

# 안전한 YOLO (rm 차단, 추천!)
claude -p --dangerously-skip-permissions --disallowedTools "Bash(rm:*)" "작업"

# 최소 권한 YOLO (읽기/수정만)
claude -p --dangerously-skip-permissions --tools "Read,Edit,Grep,Glob" "작업"
```

## 6. task 작성 팁

### 좋은 task 구조

```
## 작업: [명확한 제목]

### 경로
~/workspace/project/

### 요구사항
- 구체적인 내용 1
- 구체적인 내용 2

### 참고 파일
- docs/SPEC.md
- src/services/target.ts

### 완료 조건
- [ ] 코드 작성
- [ ] 테스트 통과
- [ ] git commit & push

### 주의사항
- 기존 XX 파일은 건드리지 마
- dev 브랜치에서 작업
```

### 나쁜 task vs 좋은 task

```
❌ "버그 좀 고쳐줘"
✅ "login.ts의 세션 만료 로직에서 refreshToken 갱신이 안 되는 버그 수정. 
   JWT 만료 시 자동 갱신 추가. 테스트 후 커밋."

❌ "API 만들어줘"
✅ "Express.js 기반 /api/users CRUD 엔드포인트 구현.
   MongoDB 연동. 입력 검증(zod). 에러 핸들링.
   테스트 작성 후 커밋 & 푸시."
```

## 7. 워크플로우 전체 흐름

```
1. 사람: "Issue #60 해결해줘"
     ↓
2. 봇 (OpenClaw):
   - exec로 Claude Code 백그라운드 실행
   - 다른 대화 계속 가능
     ↓
3. Claude Code (자동):
   - GitHub Issue 읽기
   - 코드 분석 & 수정
   - 테스트 실행
   - git commit & push
     ↓
4. wake 알림 → 봇에게 "Done!" 전달
     ↓
5. 봇: 사람에게 완료 보고
```

## 8. 자주 하는 실수

| 실수 | 증상 | 해결 |
|------|------|------|
| `-p` 빠짐 | 멈추고 아무것도 안 함 | `-p` 추가 |
| `--verbose` 빠짐 | stream-json 에러 | `--verbose` 추가 |
| `pty:true` 빠짐 | TTY 에러 | `pty:true` 추가 |
| workdir 잘못됨 | 엉뚱한 파일 수정 | 경로 재확인 |
| 브랜치 미확인 | main에 직접 커밋 | 작업 전 브랜치 체크 명시 |
