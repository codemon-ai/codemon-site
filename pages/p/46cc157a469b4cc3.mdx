# BlueMango v2 — 무통장 입금 자동화 (PayAction)

> 무통장입금 시 관리자가 수동으로 확인하던 것을 **PayAction 가상계좌로 자동화**하여, 입금 즉시(1초 내) 자동 확인됩니다.

<DocSection title="배경 — 현재 무통장 입금 프로세스의 문제" accent="sky">

### AS-IS (현재)

```
고객: 무통장입금 선택 → 회사 계좌로 입금
         ↓
관리자: 은행 앱 확인 → 입금자/금액 대조 → 수동으로 "입금 확인" 클릭
         ↓ (수시간 ~ 1일 소요)
시스템: 예약 상태 변경 → 고객에게 확인 안내
```

### 문제점

| 문제 | 영향 |
|------|------|
| **야간/주말 입금 확인 지연** | 금요일 밤 입금 → 월요일에야 확인 → 고객 불만 |
| **수작업 실수 가능** | 입금자명/금액 수동 대조 → 오확인, 누락 위험 |
| **관리자 업무 부담** | 매일 입금 확인이 반복 업무 → 다른 업무 시간 감소 |
| **고객 문의 증가** | "입금했는데 언제 확인되나요?" 문의 빈번 |
| **입금 기한 관리 불가** | 미입금 건 추적/만료 수동 관리 |

### TO-BE (변경 후)

```
고객: 무통장입금 선택 → 주문별 고유 가상계좌 안내 → 입금
         ↓ (1초 이내)
PayAction: 입금 감지 → 웹훅으로 v2 서버에 알림
         ↓ (자동)
시스템: 입금 확인 → 예약 상태 변경 → 고객에게 자동 알림
```

</DocSection>

### 관리자 화면 변경

- **결제 관리** 목록에서 무통장입금 건의 상태가 실시간 업데이트
- **자동 확인 이력** 조회 가능 (언제 입금됐는지, PayAction 응답 시간 등)
- 수동 확인 기능도 **백업으로 유지** (PayAction 장애 시 사용)

<DocSection title="기술 흐름 (비개발자 수준)" accent="sky">

### 간단히 말하면

1. 고객이 무통장입금을 선택하면 → **그 주문만을 위한 고유 계좌번호**가 만들어짐
2. 고객이 그 계좌로 입금하면 → PayAction이 **1초 안에 감지**
3. PayAction이 v2 서버에 "입금됐어요" 알려줌 → **자동으로 확인 처리**
4. 고객에게 "입금 확인되었습니다" 알림 자동 발송


<DocSection title="리스크 & 대응" accent="amber">

| 리스크 | 발생 가능성 | 대응 방안 |
|--------|-----------|----------|
| **PayAction 서비스 장애** | 낮음 | 기존 수동 확인으로 즉시 폴백 (관리자 화면에 수동 확인 버튼 유지) |
| **부분 입금** | 가끔 | 미지원 — 전액 일치만 확인. 부분 입금 시 관리자에게 알림 → 수동 처리 |
| **입금 기한 초과** | 가끔 | 자동 주문 취소 또는 관리자 알림 (설정에 따라) |
| **가상계좌 발급 실패** | 드묾 | 회사 고정 계좌 안내로 폴백 (기존 방식) |
| **웹훅 수신 실패** | 드묾 | 주기적 폴링으로 보완 (5분마다 미확인 건 재조회) |

</DocSection>

<DocSection title="타임라인 (예상)" accent="emerald">

| 단계 | 기간 | 선행조건 | 내용 |
|------|------|---------|------|
| 1. PayAction 계약 | ~1주 | 비용 확인 | 서비스 계약 + API Key 발급 |
| 2. 가상계좌 연동 | ~3일 | 단계 1 | 가상계좌 발급/조회 API 연동 |
| 3. 웹훅 처리 | ~2일 | 단계 2 | 입금 알림 수신 + 자동 상태 변경 |
| 4. 관리자 화면 | ~2일 | 단계 3 | 무통장 자동 확인 이력 조회 화면 |
| 5. 알림 연동 | ~1일 | 알림 시스템 | 입금 확인 시 이메일/알림톡 발송 |
| 6. 테스트 + QA | ~3일 | 단계 2~5 | E2E 테스트 (입금→확인→알림) |

**총 예상: 약 2~3주** (PayAction 계약 완료 시점부터)

> 알림 시스템 이관(이메일/알림톡)과 **병렬 진행 가능**

</DocSection>
