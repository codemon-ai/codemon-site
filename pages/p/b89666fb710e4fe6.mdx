# BlueMango v2 — 정보 구조 (IA)

> **작성일**: 2026-02-27
> **대상**: COO, PM

## 한 줄 요약

BlueMango v2는 **고객 사이트 42개 페이지**(14개 카테고리 상품 탐색 → 예약 → 결제 → 바우처)와 **어드민 사이트 35개 화면**(8개 메뉴 그룹)으로 구성되며, v1 대비 어드민 화면 수를 절반 이하로 줄이면서 장바구니·일정관리·정산 등 신규 기능을 추가합니다.

## 고객 사이트 구조

<DocSection title="하단 탭 (GNB)" accent="sky">

고객 사이트의 하단에 항상 노출되는 4개 탭입니다.

| 순서 | 탭 | 설명 |
|:---:|------|------|
| 1 | **홈** | 메인 화면. 추천 상품, 카테고리 바로가기, 프로모션 배너 |
| 2 | **검색** | 카테고리별 상품 검색 및 필터링 |
| 3 | **일정** | 예약한 여행 일정을 캘린더로 관리 **(v2 신규)** |
| 4 | **마이페이지** | 예약 내역, 포인트, 프로필, 알림 등 개인 메뉴 |

</DocSection>

<DocSection title="상품 카테고리 (14개)" accent="sky">

| # | 카테고리 | # | 카테고리 |
|:---:|------|:---:|------|
| 1 | 호텔 | 8 | 해양투어 |
| 2 | 풀빌라 | 9 | 가이드 |
| 3 | 골프텔 | 10 | 스파/마사지 |
| 4 | 골프장 | 11 | 레스토랑 |
| 5 | 차량 | 12 | 쇼/입장권 |
| 6 | 단독투어 | 13 | 공항패스트트랙 |
| 7 | 지상투어 | 14 | 비자 |

</DocSection>

<DocSection title="주요 사용자 흐름" accent="sky">

고객이 상품을 발견하고 바우처를 받기까지의 핵심 흐름입니다.

```
홈/검색 → 카테고리 선택 → 상품 목록 → 상품 상세
  → 옵션 선택 → 장바구니 담기 (v2 신규)
    → 주문서 작성 → 결제 → 예약 확정 → 바우처 수령
```

**v2에서 달라진 점:**

- **장바구니/주문/결제 흐름 추가** — v1은 상품 하나씩 바로 예약, v2는 여러 상품을 장바구니에 담아 한번에 결제 가능
- **일정관리 신규** — 예약 확정된 상품이 자동으로 캘린더에 반영, 여행 전체 일정을 한눈에 확인
- **공개 인보이스/바우처** — 고유 링크로 로그인 없이 인보이스/바우처 확인 가능 (카카오톡 등으로 공유 가능)

</DocSection>

<DocSection title="마이페이지 구성" accent="sky">

| 메뉴 | 설명 |
|------|------|
| 예약관리 | 진행 중/완료/취소 예약 내역 조회 및 상세 확인 |
| 포인트 | 보유 포인트 조회, 적립/사용 내역 |
| 프로필 | 개인정보 수정, 소셜 계정 연동 관리 |
| 알림 | 예약 상태 변경, 프로모션 등 알림 목록 |
| 공지사항 | 서비스 공지, 업데이트 안내 |
| 여행팁 | 태국 여행 관련 유용한 정보 콘텐츠 |

</DocSection>
