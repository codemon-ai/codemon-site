<DocSection title="주차별 상세" accent="sky">

### Week 1 — PRD 확정 + 외부 계약 (3월 수요일까지)

| 항목 | 내용 |
|------|------|
| PRD/TC 확정 | 전달된 8개 PRD 문서 검토 + 결정사항 확정 |
| PG 계약 | KG이니시스 가맹점 계약 진행 (운영팀) |
| PayAction 계약 | PayAction 서비스 계약 + 비용 확인 (운영팀) |
| 마이그레이션 범위 | **회원만** v1→v2 이관 합의 (주문/결제는 제외) |
| 알림 설정 | Mailgun DNS 설정 + BloomAI 크레덴셜 확인 (운영팀) |

> **마이그레이션 범위**: 회원 데이터만 이관하기로 합의. 기존 주문/결제 데이터는 v1에서 조회 가능하도록 유지.

### Week 2~3 — 미착수 전체 개발 + TC 100%

- 8개 미착수 항목 순차 개발
- 항목별 테스트케이스(TC) 작성 + 100% PASS 확인
- **TC 기반 완료 판정** — TC 전체 PASS가 "완료"의 기준

### Week 4 — 인간 QA + 버그 수정

| 항목 | 내용 |
|------|------|
| QA 주체 | 클라이언트(운영팀) 직접 테스트 |
| QA 기준 | TC 기반 시나리오 + 자유 탐색 테스트 |
| 버그 수정 | QA에서 발견된 버그 즉시 수정 |
| 완료 조건 | TC 기반 완료가 우선, QA 버그는 후속 수정 |

> **핵심 원칙**: TC 100% PASS가 개발 완료 기준. QA에서 나오는 버그는 TC에 없는 엣지 케이스이므로 별도 수정.

### Week 5 (4월 1주) — 소프트오픈 준비

| 항목 | 내용 |
|------|------|
| 도메인 | bluemango-travel 기존 도메인과 **듀얼 도메인** 운영 |
| 방식 | 새 도메인으로 v2 접속, 기존 도메인은 v1 유지 |
| 소프트오픈 결정 | 클라이언트 최종 확인 후 오픈 여부 결정 |

### Week 6 (4월 2주) — 소프트오픈

| 항목 | 내용 |
|------|------|
| 도메인 전환 | 기존 bluemango-travel 도메인을 v2로 연결 |
| 데이터 | 회원 마이그레이션 실행 |
| TC 검증 | 프로덕션 환경에서 TC 100% 재확인 |
| v1 보존 | v1은 별도 URL로 접근 가능 (기존 데이터 조회용) |

</DocSection>

## 비용

<DocSection title="프로젝트 비용" accent="amber">

|------|------|------|
| **총 비용** | **1,000만원** | 잔여 개발 전체 (협의 시 800만원 가능) |

### 포함 범위

- 미착수 8개 기능 전체 개발

- QA 기간 버그 수정
- 소프트오픈 + 도메인 전환
- 회원 데이터 마이그레이션 (v1 → v2)

### 미포함 (별도 협의)

- v1 주문/결제 데이터 마이그레이션
- 모바일 앱 개발
- 다국어(영어) 지원
- AI 추천 시스템
