# 시스템 인프라 변경 안내 — PG 전환 + 알림 시스템

> 작성일: 2026-02-27 | 대상: COO, PM | 문서 유형: PRD

## 1. 한 줄 요약

결제 PG사를 토스페이먼츠에서 KG이니시스로 전환하고, v1에서 운영 중인 이메일/알림톡 기능을 v2로 이관하며 PDF 생성을 추가합니다.

## 2. PG사 변경 (토스페이먼츠 → KG이니시스)

### 왜 바꾸는가

| 항목 | 토스페이먼츠 (현재) | KG이니시스 (변경) |
|------|-------------------|------------------|
| 수수료 | 상대적 높음 | 태국 여행업 특화 조건, 더 유리 |
| 정산 주기 | 표준 | 더 빠른 정산 조건 |
| 계약 상태 | 기존 계약 | 계약 완료, 카드사 심사 진행 중 |

### 고객 경험 변화

- 결제 화면 디자인이 **약간 달라집니다** (이니시스 결제창)
- **결제 과정 자체는 동일합니다** — 카드 선택 → 결제 → 완료
- 지원 결제 수단: 카드 결제 (동일)
- 영수증/결제 내역: 이니시스 브랜드로 표시

### 관리자 변화

- 환불 처리 방법: **동일** (관리자 화면에서 환불 버튼 클릭)
- 내부적으로 이니시스 API를 통해 환불 처리
- PG 로고/명칭이 이니시스로 변경

### 필요 조건

| 항목 | 상태 | 설명 |
|------|------|------|
| 이니시스 계약 | 완료 | MID(가맹점 ID) 발급 완료 |
| 카드사 심사 | 진행 중 | 심사 승인 후 운영 결제 가능 |
| Vercel IP 확인 | 확인 필요 | 정산 대사(정산 파일 다운로드) 시 고정 IP 필요 여부 이니시스에 문의 필요 |

### 리스크 & 대응

| 리스크 | 대응 |
|--------|------|
| 카드사 심사 지연 | 토스 병행 운영으로 서비스 중단 없음 |
| 이니시스 결제 장애 | **환경변수 1개 변경으로 토스로 즉시 롤백** (기존 토스 코드 보존) |
| 정산 대사 IP 이슈 | 이니시스 담당자에게 사전 확인, 필요 시 별도 방안 마련 |

## 3. 알림 시스템 정비

### v1 → v2 이관 현황

| 항목 | 현재 상태 |
|------|----------|
| 이메일 발송 | **v2 연결 미완** — v1에서 Resend로 구현되어 있으나 v2 발송 연결이 안 되어 있음 |
| 알림톡 (카카오) | **v2 이식 필요** — v1에서 BloomAI로 운영 중, v2에 이식 필요 |
| 인보이스/바우처 PDF | **PDF 추가 예정** — v1에서 웹 페이지로 제공, v2에서 PDF 다운로드 추가 예정 |

> v1에서 운영 중인 기능을 v2로 이관하는 작업입니다. 현재 v2에서는 자동 알림이 연결되지 않은 상태이며, v1의 검증된 기능을 이식합니다.

### 변경 후

| 항목 | 변경 내용 |
|------|----------|
| 이메일 | 각 단계별 자동 발송 (접수/결제/인보이스/바우처/취소) |
| 알림톡 | v1에서 사용하던 카카오 알림톡 복원 (이메일과 동시 발송) |
| PDF | 인보이스/바우처 PDF 다운로드 링크 제공 (웹 페이지도 유지) |

### 이메일 변경 — Resend → Mailgun

| 항목 | 현재 (Resend) | 변경 (Mailgun) |
|------|--------------|----------------|
| 비용 | 유료 | 월 1,000건 무료, 이후 과금 |
| 도메인 | 설정 안 됨 | `mg.bluemango-travel.com` 설정 필요 |
| 호환성 | 문제 없음 | Vercel 서버리스 호환 |

**운영팀 필요 작업**: Mailgun 계정 생성 + 도메인 DNS 설정 (아래 §4 참조)

### 알림톡 복원 — BloomAI (루나소프트)

- v1에서 사용하던 **BloomAI(루나소프트) 서비스를 그대로 사용**
- 기존 계정/크레덴셜 재활용 (신규 가입 불필요)
- 활성 템플릿 5개:

| 시점 | 템플릿 | 내용 |
|------|--------|------|
| 예약 접수 | 접수 완료 | "○○님의 ○○ 접수가 완료되었습니다" |
| 인보이스 발행 | 예약 확정 | "인보이스가 발송되었습니다" + 인보이스 링크 |
| 바우처 발행 | 바우처 발송 | "바우처가 발송되었습니다" + 바우처 링크 |
| 취소 요청/완료 | 취소 접수 | "취소 요청이 접수되었습니다" |
| 문의 답변 | 답변 알림 | "질문하신 내용에 답변이 등록되었습니다" |

- 알림톡 전송 실패 시 **SMS 자동 폴백** (v1과 동일)

### 인보이스/바우처 PDF

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 웹 페이지 | 공개 링크로 접근 가능 (유지) | 동일 (유지) |
| PDF | 없음 | PDF 다운로드 링크 추가 |
| 이메일 첨부 | 없음 | 이메일에 PDF 다운로드 링크 포함 |

- 기존 웹 링크(인보이스/바우처 페이지)는 **그대로 유지**
- PDF는 추가 옵션으로 제공 (고객이 필요할 때 다운로드)

### 알림 트리거 시점 정리

| 단계 | 이메일 | 알림톡 | PDF |
|------|--------|--------|-----|
| 예약 접수 | 접수 확인 이메일 | 접수 완료 알림톡 | — |
| 결제 완료 | 결제 확인 이메일 | — (v1에서도 미발송) | — |
| 인보이스 발행 | 인보이스 이메일 (링크 포함) | 예약 확정 알림톡 (링크 포함) | 인보이스 PDF |
| 바우처 발행 | 바우처 이메일 (링크 포함) | 바우처 발송 알림톡 (링크 포함) | 바우처 PDF |
| 취소 요청 | 취소 접수 이메일 | 취소 접수 알림톡 | — |
| 취소 완료 | 취소 완료 이메일 | 취소 접수 알림톡 | — |

> 이메일과 알림톡은 **동시 발송**됩니다. 하나가 실패해도 다른 채널은 정상 발송됩니다.

## 4. 환경 설정 필요 사항 (운영팀 액션 아이템)

### Mailgun 도메인 설정

1. Mailgun 계정 생성 (https://www.mailgun.com)
2. 발송 도메인 추가: `mg.bluemango-travel.com`
3. DNS 레코드 5개 추가 (도메인 호스팅 업체에서):
   - SPF (TXT 레코드)
   - DKIM (TXT 레코드)
   - 트래킹 (CNAME 레코드)
   - 수신 (MX 레코드 2개)
4. Mailgun 대시보드에서 "DNS 검증" 클릭
5. API Key 발급 → 개발팀에 전달

> DNS 설정 후 검증까지 보통 수 분~최대 48시간 소요

### BloomAI (알림톡) 크레덴셜

- v1에서 사용하던 **동일한 계정 정보** 사용
- 필요한 정보 (v1 서버에서 확인):
  - BloomAI User ID
  - BloomAI API Key
  - BloomAI Sender Key
  - SMS 폴백 발신번호

> 신규 가입이나 추가 설정은 불필요합니다.

### 이니시스 관련

- 카드사 심사용 자료 제출 (사업자등록증, 결제 화면 스크린샷 등)
- 정산 대사 IP 관련 이니시스 담당자 확인

## 5. 타임라인

| 단계 | 기간 | 선행조건 | 내용 |
|------|------|---------|------|
| 1. Mailgun 설정 | ~2일 | Mailgun 계정 | DNS 설정 + 검증 |
| 2. 이메일 정상화 | ~1일 | 단계 1 | 이메일 서비스 교체 + 자동 발송 연결 |
| 3. 알림톡 복원 | ~2일 | BloomAI 크레덴셜 | v1 알림톡 기능 v2 이식 |
| 4. PDF 생성 | ~2일 | 없음 (독립) | 인보이스/바우처 PDF 다운로드 |
| 5. 이니시스 연동 | ~3일 | 카드사 심사 | PG 결제 모듈 교체 |
| 6. 테스트 + QA | ~2일 | 단계 2~5 | 전체 알림/결제 E2E 테스트 |
| 7. 운영 전환 | ~1일 | 단계 6 | PG 전환 (환경변수 변경) |

**총 예상: 약 2~3주** (Mailgun 설정 + BloomAI 크레덴셜 확보 시점부터)

> 이메일/알림톡 정상화(단계 2~3)와 PG 전환(단계 5)은 **병렬 진행 가능**

## 6. 결정 필요 사항

| # | 질문 | 개발팀 추천 |
|---|------|------------|
| N-01 | Mailgun 계정을 누가 생성하고 관리할 것인가? | 운영팀이 생성, API Key를 개발팀에 전달 |
| N-02 | 알림톡 발송 시간 제한을 둘 것인가? (예: 야간 미발송) | 없음 — 여행 예약 특성상 시간 무관 |
| N-03 | PDF 파일에 회사 로고/브랜딩을 포함할 것인가? | 포함 추천 — 인보이스/바우처에 블루망고 로고 |
