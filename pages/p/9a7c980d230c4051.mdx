## 스킬 + 자동화 = 진짜 비서

1편에서 만든 스킬은 내가 "해줘" 해야 움직였다.
근데 진짜 비서는? **시키지 않아도 할 일을 한다.**

Claude Cowork에는 **스케줄 태스크**가 있다.
"매일 아침 8시에 이거 해" 같은 반복 작업을 걸어둘 수 있다.

## 자동화 스킬 1: 아침 브리핑

### 매일 아침, 켜자마자 오늘 할 일이 정리되어 있다면?

```
이름: morning-briefing
설명: 매일 아침 8시에 자동 실행.
      오늘 날짜, 요일, 날씨를 확인하고
      캘린더 일정을 정리하고
      어제 작성한 메모 중 미완료 항목을 알려준다.
      결과를 "오늘의 브리핑" 파일로 저장한다.

스케줄: 매일 08:00
```

<!-- SCREENSHOT-2: morning-briefing 스킬을 실제로 생성하고 스케줄 08:00으로 설정한 화면 -->

**실행 결과 예시:**

```
📋 2026년 3월 4일 화요일 브리핑

☀️ 서울 맑음 3°C → 12°C

📅 오늘 일정:
  10:00 팀 미팅 (Zoom)
  14:00 클라이언트 콜
  16:00 코드 리뷰

📌 어제 미완료:
  - 제안서 초안 마무리
  - 디자인 피드백 전달

💡 오늘 집중할 것: 제안서 마무리 → 14시 콜 전에 끝내기
```

<!-- SCREENSHOT-3: 아침 브리핑 스킬이 실제로 자동 실행된 결과 채팅 화면 (위 예시와 비슷한 실제 결과) -->

아침에 눈 뜨고 커피 한 잔 하면서 이거 한 번 훑으면 끝.
날씨 앱, 캘린더 앱, 메모 앱 따로따로 확인하던 시간 → **0분**.

## 자동화 스킬 2: 파일 자동 정리

### Downloads 폴더, 지금 몇 개 쌓여 있나?

다운로드 폴더 열어보면 PDF, 이미지, 엑셀, 압축파일이 뒤섞여 있다.
정리해야지 정리해야지 하면서 안 한 지 3개월.

<!-- SCREENSHOT-4: 실제 지저분한 Downloads 폴더 (Before) — 본인 폴더 or 예시 폴더 -->

```
이름: file-organizer
설명: "다운로드 정리해줘" 하면 Downloads 폴더를 스캔해서
      파일을 종류별로 분류한다.
      - PDF → Documents/PDF/
      - 이미지(jpg, png) → Pictures/
      - 엑셀/CSV → Documents/Spreadsheets/
      - 나머지 → Documents/Others/
      이동 전에 목록을 보여주고 확인받은 후 실행.
      "자동 정리", "파일 정리" 하면 실행.
```

**포인트: "확인받은 후 실행"**

이 한 줄이 중요하다. AI가 멋대로 파일을 옮기면 낭패니까.
스킬에 "먼저 보여주고 → 확인받고 → 실행" 순서를 넣어두면 안전하다.

## 자동화 스킬 3: 주간 리포트 자동 생성

### 금요일마다 이번 주 뭐 했는지 정리하는 게 귀찮다면

```
이름: weekly-report
설명: 매주 금요일 17시에 자동 실행.
      이번 주에 작성한 문서, 메모, 회의록을 모아서
      주간 업무 보고서를 생성한다.
      형식:
      - 이번 주 완료한 일
      - 진행 중인 일
      - 다음 주 계획
      - 특이사항
      결과를 "주간보고_YYMMDD.md" 파일로 저장.

스케줄: 매주 금요일 17:00
```

<!-- SCREENSHOT-7: 주간 리포트 스킬이 자동 생성한 "주간보고_260304.md" 파일 내용 -->

**금요일 퇴근 전 루틴:**
- 이전: 30분~1시간 끙끙대며 주간보고 작성
- 이후: AI가 만든 초안 5분 검토 → 제출

---

## 스킬 조합의 힘: 체이닝

스킬 하나하나도 유용하지만, **스킬끼리 연결**하면 더 강력해진다.

예시:

```
아침 브리핑 (08:00)
    ↓ 오늘 일정에 "회의" 있으면
회의록 스킬 대기
    ↓ 회의 끝나고 내용 입력하면
회의록 자동 정리
    ↓ 액션 아이템 추출
주간 리포트에 자동 반영 (금요일)
```

한 번 세팅하면 월~금 업무 루틴이 **반자동**으로 돌아간다.

이게 "AI를 도구로 쓰는 것"과 "AI를 비서로 쓰는 것"의 차이다.

---

## 자동화 설정 시 주의할 점

### 1. 확인 단계를 넣어라
파일 삭제, 이동, 메일 발송 같은 건 반드시 "먼저 보여주고 확인" 단계를 넣자.
AI가 실수하면 되돌리기 어렵다.

### 2. 작게 시작해라
처음부터 10개 스킬을 만들지 말고, **딱 하나**부터 시작.
아침 브리핑 하나만 2주 써보고, 그 다음에 추가하자.

### 3. 설명을 계속 다듬어라
결과가 기대와 다르면? description을 고치면 된다.
사람한테 "아, 그게 아니라 이렇게 해줘" 하는 것처럼.

## 실제 사례: 코드몬 팀의 하루

나(코드몬)는 AI와 함께 일하는 1인 개발자다.
실제로 이런 스킬들을 조합해서 쓰고 있다:

| 시간 | 스킬 | 하는 일 |
|------|------|---------|
| 08:00 | 아침 브리핑 | 날씨 + 일정 + 미완료 |
| 수시 | 블로그 드래프트 | 아이디어 → 초안 생성 |
| 수시 | 회의록 정리 | 메모 → 구조화 |
| 17:00 | 서비스 상태 체크 | 서버 26개 모니터링 |
| 금 17:00 | 주간 리포트 | 이번 주 자동 요약 |

<!-- SCREENSHOT-8: 실제 코드몬 작업 환경 — Claude Code 터미널 or PM2 대시보드 (선택) -->

26개 서비스를 혼자 운영하면서도 버틸 수 있는 이유.
반복 업무는 AI한테 넘기고, 나는 **판단이 필요한 일**에 집중한다.

## 인스타/스레드용 요약

### 스레드 버전 (500자 이내)

AI한테 "매일 아침 8시에 이거 해" 걸어두면?

진짜로 합니다.

Claude Desktop에 '스케줄 태스크'라는 게 있음.
스킬 + 스케줄 = 자동 비서.

내가 실제로 쓰는 3가지:

1️⃣ 아침 브리핑 (08:00 자동)
→ 날씨 + 일정 + 어제 미완료 한 장 정리

2️⃣ 파일 정리
→ "다운로드 정리해줘" 한 마디에 종류별 분류

3️⃣ 주간 리포트 (금 17:00 자동)
→ 이번 주 한 일 자동 요약 → 5분 검토 → 제출

핵심은 '확인 단계'를 넣는 것.
AI가 멋대로 파일 옮기면 안 되니까.
"먼저 보여줘 → OK하면 실행" 이 한 줄이 중요.

하나만 2주 써보세요. 습관 되면 못 끊음.
