# AI에게 '스킬'을 가르치면 무슨 일이 벌어질까

> 시리즈: AI한테 일 가르치기 (1/3)

---

## 매번 같은 말 반복하고 있다면

"이거 요약해줘."
"어제 형식이랑 똑같이 해줘."
"아까 말한 그 양식으로 다시 만들어줘."

ChatGPT든 Claude든, AI 쓰면서 이런 말 하루에 몇 번씩 하고 있지 않나? 매번 처음 만난 사람한테 업무 인수인계하는 느낌. 피곤하다.

그런데 만약 AI한테 **"이건 이렇게 하는 거야"를 한 번만 가르쳐두면**, 다음부터는 말 안 해도 알아서 하면 어떨까?

이게 바로 **스킬(Skill)**이다.

---

## 스킬이 뭔데?

한 마디로: **AI에게 가르쳐둔 업무 매뉴얼**.

사람으로 치면 이런 거다:
- 신입한테 "회의록은 이 양식으로 써" → 다음부터 알아서 함
- 인턴한테 "메일 요약은 이렇게 해" → 매번 안 알려줘도 됨

AI 스킬도 같다. 한 번 만들어두면:
1. **자동으로 인식** — "회의록 정리해줘" 하면 스킬이 알아서 발동
2. **일관된 결과** — 매번 양식 설명 안 해도 됨
3. **시간 절약** — 반복 설명 0초

코딩? 필요 없다. 한국어로 쓰면 된다.

## 스킬의 핵심: description

스킬에서 제일 중요한 건 **설명(description)**이다.

AI는 이 설명을 읽고 "아, 이 상황에서 이 스킬을 써야 하는구나" 판단한다.

**잘 쓴 예:**
```
YouTube 영상 URL을 받으면 영상 내용을 요약해서
제목, 핵심 포인트 3개, 한 줄 감상으로 정리한다.
"이거 요약해줘", "영상 정리해줘" 하면 자동으로 실행.
```

**못 쓴 예:**
```
유튜브 스킬
```

<!-- SCREENSHOT-5: 위 두 description을 나란히 비교하는 화면 (Cowork 스킬 편집 UI) — 없으면 텍스트 비교 이미지로 대체 -->

차이가 느껴지나? 잘 쓴 설명은:
- **언제 쓰는지** 알려줌 (YouTube URL 받으면)
- **뭘 하는지** 구체적 (제목, 핵심 3개, 한 줄 감상)
- **어떤 말에 반응하는지** 명시 ("이거 요약해줘")

사람한테 업무 지시할 때도 마찬가지 아닌가. "이거 해줘"보다 "이 형식으로 이렇게 해줘"가 결과가 좋다.

## 실전: 바로 써먹을 스킬 3개

### 1. 회의록 정리 스킬

```
이름: meeting-notes
설명: 회의 내용을 텍스트로 받으면 다음 형식으로 정리한다.
      - 날짜/참석자
      - 논의 사항 (번호 매김)
      - 결정 사항
      - 다음 액션 아이템 (담당자 + 기한)
      "회의록 정리해줘", "미팅 노트 만들어줘" 하면 실행.
```

### 2. 이메일 초안 스킬

```
이름: email-draft
설명: 상황을 설명하면 비즈니스 이메일 초안을 작성한다.
      톤은 정중하되 간결하게. 한국어/영어 지정 가능.
      제목 + 본문 + 마무리 인사 포함.
      "메일 써줘", "답장 초안 만들어줘" 하면 실행.
```

### 3. 블로그 아이디어 브레인스토밍

```
이름: blog-ideas
설명: 주제나 키워드를 주면 블로그 글 아이디어 5개를 제안한다.
      각 아이디어마다 제목, 핵심 메시지, 예상 반응을 포함.
      "블로그 아이디어 줘", "이 주제로 뭐 쓸까" 하면 실행.
```
