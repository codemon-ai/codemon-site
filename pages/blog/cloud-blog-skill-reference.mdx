# Cloud Blog Skill 분석 — Claude Code 블로그 자동화의 끝판왕?

> `/blog write`로 3000자+ 블로그를 5분 만에. SVG 차트, 무료 이미지, 팩트체크까지. 이게 진짜 되는 걸까?

![5단계 파이프라인으로 자동화되는 블로그 발행 흐름](/images/blog/cloud-blog-skill-reference/hero.png)

## 영상 하나가 눈에 꽂혔다

E Marketing Hub의 [Cloud Blog Skill 데모 영상](https://youtu.be/AeLC4iutG8w)을 봤다. Claude Code에 `/blog write` 슬래시 커맨드를 치면 주제, 독자, 분량, 포맷을 선택하고 — 5분 만에 3000자짜리 블로그가 나온다.

솔직히 처음엔 "또 AI 블로그 자동화 영상이네" 했다. 그런데 디테일이 달랐다.

**이건 단순한 "GPT한테 글 써달라고 하기"가 아니다.** 5단계 파이프라인이 있고, 각 단계마다 다른 전략이 붙어있다. Tech Lead 관점에서 분석해봤다.

## 5단계 파이프라인 분석

Cloud Blog Skill은 다음 순서로 동작한다:

### 1단계: 템플릿 선택

주제를 입력하면 적절한 블로그 템플릿을 자동 선택한다. 리스트형, 가이드형, 비교형, 케이스스터디형 등. 구조가 먼저 잡히니까 글이 산만해지지 않는다.

### 2단계: 리서치 자동화

![웹 검색 결과에서 데이터를 추출하고 정리하는 리서치 자동화](/images/blog/cloud-blog-skill-reference/research.png)

Brave Search API로 웹을 검색하고, **Tier 1 소스(정부기관, 교육기관, 공식 문서)를 우선 수집**한다. 이게 핵심이다. 아무 블로그나 긁어오는 게 아니라, 신뢰도 높은 소스를 먼저 가져온다.

2025년 기준 영어 콘텐츠의 52%가 AI로 생성되는 시대다. 팩트체크 없는 AI 글은 AI 글을 재생산하는 순환에 빠진다. Tier 1 소스 우선 수집은 이 문제를 정면으로 해결한다.

### 3단계: 아웃라인 생성

리서치 결과를 바탕으로 아웃라인을 자동 생성한다. 여기서 SEO 키워드 배치, 섹션 구조, 예상 분량까지 계획된다.

### 4단계: 본문 작성 + 이미지/차트

Pixabay, Pexels에서 무료 이미지를 자동 크롤링해서 삽입한다. 여기까지는 흔하다. 진짜 인상적인 건 **SVG 차트 자동 생성**이다. 데이터가 있으면 인라인 SVG로 시각화를 만들어 넣는다.

![SVG 차트와 데이터 시각화가 3D 공간에 떠있는 모습](/images/blog/cloud-blog-skill-reference/charts.png)

### 5단계: 품질 체크

완성된 글에 자동으로 품질 점수를 매긴다. 데모에서 72/100이 나왔고, 인간이 리뷰해서 수정하면 90까지 올라갔다. SEO 점수, 가독성, 팩트 정확도를 종합 평가한다.

## 우리 blog-publish 스킬과 비교

![두 시스템의 비교](/images/blog/cloud-blog-skill-reference/comparison.png)

우리도 블로그 자동 발행을 하고 있다. 13개월간 운영한 blog-publish 스킬과 비교해봤다.

| 항목 | Cloud Blog Skill | 우리 blog-publish |
|------|-----------------|-------------------|
| **리서치** | Brave Search 자동 (Tier 1 우선) | Gemini Flash 리서치 워커 |
| **이미지** | Pixabay/Pexels 크롤링 | Nano Banana Pro AI 생성 |
| **차트** | SVG 자동 생성 ✅ | 없음 ❌ |
| **팩트체크** | Tier 1 소스 자동 우선 | 수동 (Brave Search) |
| **품질 채점** | 자동 점수 (72/100) | 없음 |
| **발행 대상** | HTML / WordPress | MDX → Vercel (자동 배포) |
| **이미지 품질** | 무료 스톡 (범용적) | AI 생성 (브랜드 일관성) |
| **파이프라인** | 슬래시 커맨드 1회 | 드래프트 → 이미지 → MDX → git push |

**솔직한 평가:** 각각 장단이 있다.

Cloud Blog Skill은 **원스톱 자동화**에 강하다. 한 번 커맨드로 끝까지 간다. 반면 우리 시스템은 **커스터마이징과 브랜드 일관성**에 강하다. AI 생성 이미지로 통일된 비주얼을 유지하고, MDX + Vercel 파이프라인으로 개발자 친화적 배포를 한다.

## 배울 점 3가지

### 1. SVG 차트 자동 생성

이건 확실히 우리한테 없는 기능이다. 데이터가 있는 글에서 인라인 차트를 자동으로 넣어주면 체류 시간이 늘어난다. Mermaid나 D3.js 기반으로 구현할 수 있겠다.

### 2. 품질 자동 채점

글을 쓰고 "이거 괜찮나?" 하는 건 사람이나 AI나 마찬가지다. SEO 점수, 가독성 지수, 키워드 밀도를 자동으로 체크하는 파이프라인은 꼭 추가하고 싶다.

### 3. Tier 1 소스 우선 리서치

우리의 리서치는 아직 수동에 가깝다. Brave Search를 쓰긴 하지만, 소스 신뢰도를 자동으로 분류하진 않는다. 도메인 기반 Tier 분류는 구현 난이도 대비 효과가 높다.

## 개선 로드맵

![개선과 혁신 — 회로 패턴 전구와 상승 화살표](/images/blog/cloud-blog-skill-reference/improvement.png)

Cloud Blog Skill에서 영감받아 우리 시스템에 추가할 기능들:

1. **SVG 차트 자동 삽입** — Mermaid 문법으로 데이터 시각화 (난이도: 중)
2. **품질 자동 채점** — SEO 점수 + 가독성 + 키워드 밀도 체크 (난이도: 하)
3. **Pixabay/Pexels 폴백** — AI 이미지 생성 실패 시 무료 스톡 자동 삽입 (난이도: 하)
4. **소스 신뢰도 자동 분류** — .gov, .edu, 공식 문서 우선 (난이도: 하)

핵심은 **인간 작성 글이 유기 트래픽에서 5.44배 높다**는 통계다. AI가 초안을 만들되, 인간의 관점과 경험이 들어가야 한다. Cloud Blog Skill도 "72점 → 인간 리뷰 → 90점" 흐름을 강조한다.

완전 자동화가 아니라 **반자동화 + 인간 감수**. 이게 2026년 AI 블로그 작성의 정답에 가장 가깝다.
