## 시스템 아키텍처

![WikiCBOW System Architecture](/yonsei-wikicbow/figures/architecture.svg)

**Training** (왼쪽): Vault → 3 pair sources (intra 231k · wiki 180 · cat 97k) → λ-mixed batches → PyTorch CBOW with NS loss → learned W₁.
**Retrieval** (오른쪽): query and notes both pooled via mean or TF-IDF → cosine top-k.

두 phase 가 학습된 W₁ 임베딩 하나를 공유합니다.

## 기여 포인트

1. **3-소스 pair generation 통합 형식화** — intra / wiki / cat 을 하나의 NS 손실 안에 stratified mixing 으로 통합한 단일 모델 (PyTorch 직접 구현)
2. **TF-IDF weighted pooling + 학습된 임베딩 결합** — 정보 검색 고전 기법과 분산 표현의 명시적 결합 (B6)
3. **그래프 신호의 ablation** — wikilink/category 각 신호가 retrieval 성능에 미치는 한계 효과를 P@k 기준으로 정량화
4. **개인 vault 케이스 스터디** — 단일 사용자 노트 컬렉션 위 personal embedding의 실용성·한계 보고
