# 신호가 오면 자동 주문하는 나만의 투자 봇

> ⚠️ **모의(페이퍼) 투자 전용 · 교육/프로토타입용 · 투자 조언 아님.** 모든 코드는 `paper=True`(가짜 돈)로 동작합니다.

> 📄 **[전체 강의 자료 페이지 보기 →](/slides/lecture-trading-bot.html)** — 세로 스크롤 풀 버전(구조도·준비물·커리큘럼·핵심 코드)

> 📌 본 페이지는 **강의 개요(목차)** 입니다.

## 개요

| 항목 | 내용 |
|------|------|
| 제목 | 신호가 오면 자동 주문하는 나만의 투자 봇 |
| 형식 | 핸즈온 (2시간 / 120분) · 입문 |
| 대상 | 코딩 입문자 + 자동매매 원리가 궁금한 분 |
| 강사 | 코드몬 (codemon.ai) |
| 구조 | TradingView 알림 → 웹훅(Flask) → Alpaca 페이퍼 주문 → 텔레그램 알림 + Streamlit 대시보드 |
| 전제 | **모의(페이퍼) 전용**, 실제 돈 사용 안 함 |

## 이런 분에게 추천합니다

- "자동매매 봇"이 실제로 **어떻게 동작하는지** 원리를 손으로 만들어 보고 싶은 분
- 신호 → 주문 → 알림 → 대시보드로 이어지는 **파이프라인 전체**를 경험하고 싶은 분
- 돈을 잃을 걱정 없이 **모의계좌로 안전하게** 코드를 돌려보고 싶은 분

## 핵심 성과

> 2시간 후, **신호를 받으면 스스로 모의 주문을 넣고 폰으로 알려주는 봇 1개**를 손에 쥡니다.

- 내 코드가 Alpaca 페이퍼에 실제로 주문을 넣는 경험
- TradingView(또는 `curl`) 신호 → 웹훅 → 자동 주문까지 동작하는 봇
- 텔레그램 체결 알림 + Streamlit 대시보드

## 준비물

| 도구 | 역할 | 비용 |
|------|------|------|
| **Alpaca 페이퍼** | 모의 주문 브로커 (가입 → Paper 계정 → API 키, Secret 1회 노출) | 무료 |
| **TradingView** | 신호 발생원. **웹훅은 유료(Essential+) + 2FA 필수** — 없으면 `curl` 시뮬레이션으로 대체(전원 가능) | 선택·유료 |
| **텔레그램** | @BotFather 봇 생성 → 토큰 → chat_id | 무료 |
| **Python + 패키지** | `pip install alpaca-py flask requests streamlit` | 무료 |
| **ngrok** | 로컬 서버를 공개 HTTPS로. `ngrok http 5000` | 무료 |

> Alpaca 페이퍼·텔레그램·ngrok·Python은 전부 무료. **TradingView 웹훅만 유료**라, 수업은 **무료 `curl` 시뮬레이션을 기본**으로 진행하고 유료 TradingView가 있으면 실제 알림까지 연결합니다.

## 커리큘럼 (120분)

> 시연 / 직접 실습 중심. 매 단계 끝에 동작하는 결과 하나씩.

### 0. 큰 그림 & 안전 (10분)
전체 아키텍처, **페이퍼 전용·리스크·비용** 고지. "신호 → 웹훅 → 주문" 한 문장 이해.

### 1. 계정·키 준비 (20분)
Alpaca 페이퍼 가입 + 키(Secret 1회 저장) · 텔레그램 BotFather 봇+토큰+chat_id · `pip install` · (TradingView 웹훅 유료·2FA 안내)

### 2. 첫 주문 — Alpaca 페이퍼 (20분)
`TradingClient(paper=True)` → 시장가 주문 1건 → `get_account`·`get_all_positions`로 확인.

### 3. ★ 웹훅 봇 — 신호가 오면 자동 주문 (35분)
Flask `/webhook`(passphrase 검증 → 주문) → `ngrok http 5000` 공개 → **`curl`로 신호 쏘기**(또는 TradingView 알림 JSON) → 자동 주문 확인. 보안: passphrase + 발신 IP 4개 + `paper`.

### 4. 텔레그램 알림 연결 (15분)
주문 체결 시 `sendMessage`로 폰에 푸시.

### 5. 대시보드 — Streamlit (15분)
`streamlit run`으로 포트폴리오 가치·포지션·손익 표시.

### 6. 마무리 & 리스크 (5분)
백테스트 **과최적화** 경고, 실거래 위험, 다음 단계. 안전 재강조.

## 정직 포인트 (과장 없이)

- **TradingView엔 '사용자용 체결 API'가 없습니다.** 표준은 *알림 → 웹훅 → 브로커(Alpaca) API*.
- **"TradingView MCP"는 전부 비공식** 커뮤니티 제작·스크래핑 기반(ToS 리스크) — 교육/프로토타입용으로만.
- 가장 유명한 튜토리얼 레포는 **구버전 `alpaca-trade-api`** → 현행 **`alpaca-py`** 로 작성.
- 웹훅엔 서명 인증이 없으니 **passphrase + 발신 IP 화이트리스트 + `paper=True`** 로 방어.
- 이 수업의 목표는 '돈 버는 봇'이 아니라 **'파이프라인을 직접 만들어보는 것'** 입니다.
