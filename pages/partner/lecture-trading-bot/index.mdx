# 신호가 오면 자동 주문하는 나만의 투자 봇

> ⚠️ **모의(페이퍼) 투자 전용 · 교육/프로토타입용 · 투자 조언 아님.** 모든 코드는 `paper=True`(가짜 돈)로 동작합니다. 이 강의는 수익 전략이나 실거래 전환을 다루지 않고, 신호→웹훅→모의 주문 파이프라인을 이해하는 수업입니다.

> 📄 **[실습 가이드 (전체 자료) →](/slides/lecture-trading-bot.html)** — 수업 때 쓰는 코드·명령어·확인 방법을 한곳에 모아둔 자료

> 📌 본 페이지는 **강의 개요(목차)** 입니다.

## 개요

| 항목 | 내용 |
|------|------|
| 제목 | 신호가 오면 자동 주문하는 나만의 투자 봇 |
| 형식 | 핸즈온 (2시간 / 120분) · 입문 |
| 대상 | 코딩 입문자 + 자동매매 원리가 궁금한 분 |
| 강사 | 코드몬 (codemon.ai) |
| 구조 | `curl` 또는 TradingView 알림 → 웹훅(Flask) → Alpaca 페이퍼 주문 → 텔레그램 알림 + 선택 대시보드 |
| 전제 | **모의(페이퍼) 전용**, 실제 돈 사용 안 함 |

## 이런 분에게 추천합니다

- "자동매매 봇"이 실제로 **어떻게 동작하는지** 원리를 손으로 만들어 보고 싶은 분
- 신호 → 주문 → 알림 → 대시보드로 이어지는 **파이프라인 전체**를 경험하고 싶은 분
- 돈을 잃을 걱정 없이 **모의계좌로 안전하게** 코드를 돌려보고 싶은 분

## 수업이 끝나면

> 2시간 동안 신호를 보내고, 모의 주문과 텔레그램 알림까지 직접 확인합니다.

- 내 코드가 Alpaca 페이퍼에 실제로 주문을 넣는 경험
- `curl` 신호 → 웹훅 → 자동 주문까지 모두 같이 확인하는 봇
- 텔레그램 체결 알림 + 선택형 Streamlit 대시보드

## 진행 방식

| 경로 | 의미 | 목표 |
|------|------|------|
| **기본 진행** | 모두 같이 하는 무료 경로 | `curl` → Flask 웹훅 → Alpaca 페이퍼 주문 → 텔레그램 알림 |
| **TradingView 연결** | 유료 플랜과 2FA가 준비된 경우 | 실제 TradingView 알림 JSON을 웹훅에 연결 |
| **시간이 남으면** | 수업 후 더 키워볼 부분 | Streamlit 대시보드, IP allowlist, 환경변수, 로깅 보강 |

## 준비물

| 구분 | 도구 | 역할 | 비용 |
|------|------|------|------|
| **필수** | **Alpaca 페이퍼** | 모의 주문 브로커 (가입 → Paper 계정 → API 키, Secret 1회 노출) | 무료 |
| **필수** | **Python + 패키지** | `pip install alpaca-py flask requests` | 무료 |
| **필수** | **ngrok** | 로컬 Flask 서버를 공개 HTTPS URL로 연결. 실행 시 터미널에 표시되는 forwarding URL을 사용 | 무료 |
| **권장** | **텔레그램** | @BotFather 봇 생성 → 토큰 → chat_id. 체결 알림 확인에 사용 | 무료 |
| **선택** | **TradingView** | 실제 알림 신호 발생원. 웹훅은 지원 유료 플랜 + 2FA 필요, 수업일 가격표 재확인 | 선택·유료 |
| **선택** | **Streamlit** | 포트폴리오 대시보드 확장 (`pip install streamlit`) | 무료 |

> 수업은 무료 `curl` 시뮬레이션으로 진행합니다. TradingView 유료 웹훅과 Streamlit 대시보드는 준비된 분만 이어서 연결합니다.

## 커리큘럼 (120분)

> 시연보다 직접 실행에 시간을 더 둡니다. 각 단계 끝에는 화면에서 결과를 확인합니다.

### 0. 큰 그림 & 안전 (10분)
전체 아키텍처, **페이퍼 전용·리스크·비용** 고지. "신호 → 웹훅 → 주문" 한 문장 이해.

### 1. 계정·키 준비 (20분)
Alpaca 페이퍼 가입 + 키(Secret 1회 저장) · 텔레그램 BotFather 봇+토큰+chat_id · `pip install` · TradingView는 선택 연결 안내

### 2. 첫 주문 — Alpaca 페이퍼 (20분)
`TradingClient(paper=True)` → 시장가 주문 1건 → `get_account`·`get_all_positions`로 확인.

### 3. ★ 웹훅 봇 — 신호가 오면 자동 주문 (35분)
Flask `/webhook`(passphrase 검증 → 주문) → `ngrok http 5000` 공개 → **`curl`로 신호 쏘기** → 자동 주문 확인. TradingView 알림 JSON은 유료 플랜이 있는 경우만 연결. starter 코드는 passphrase 중심, 보안 강화는 IP allowlist·환경변수·rate limit로 확장.

### 4. 텔레그램 알림 연결 (15분)
주문 체결 시 `sendMessage`로 폰에 푸시.

### 5. 시간이 남으면 — Streamlit 대시보드 (15분)
`streamlit run`으로 포트폴리오 가치·포지션·손익 표시.

### 6. 마무리 & 리스크 (5분)
백테스트 **과최적화** 경고, 실거래 위험, 다음 단계. 안전 재강조.

## 이 수업의 범위

- **TradingView엔 '사용자용 체결 API'가 없습니다.** 표준은 *알림 → 웹훅 → 브로커(Alpaca) API*.
- **"TradingView MCP"는 전부 비공식** 커뮤니티 제작·스크래핑 기반(ToS 리스크) — 교육/프로토타입용으로만.
- 가장 유명한 튜토리얼 레포는 **구버전 `alpaca-trade-api`** → 현행 **`alpaca-py`** 로 작성.
- starter 코드는 **passphrase + `paper=True`** 로 시작합니다. 운영 보안은 발신 IP allowlist, 환경변수, rate limit, 주문 크기 제한을 추가해야 합니다.
- 이 수업에서 **만드는 것**은 이벤트 기반 자동 주문 파이프라인입니다. **만들지 않는 것**은 수익 전략, 백테스트 검증, 실거래 전환, 리스크 관리 시스템입니다.
