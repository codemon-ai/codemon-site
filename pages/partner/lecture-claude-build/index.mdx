## 개요

| 항목 | 내용 |
|------|------|
| 제목 | 클로드로 만드는 나만의 지식노트 + 배포되는 내 사이트 |
| 형식 | 핸즈온 + 개인교습 (4시간 / 240분) |
| 대상 | 비개발 실무자 (프롬프트·코딩 경험 무관) |
| 강사 | 코드몬 (codemon.ai) |
| 수업 결과 | ① 클로드가 읽는 내 지식노트 볼트 &nbsp; ② 인터넷에 배포된 내 사이트 URL |

## 이런 분에게 추천합니다

- AI를 써보고 싶은데 **어디서부터 시작해야 할지** 모르겠는 분
- 메모는 쌓는데 다시 안 보게 되는 분 — **AI가 읽고 정리해주는 지식노트**가 필요한 분
- 코드 한 줄 몰라도 **내 사이트를 인터넷에 띄워보고** 싶은 분
- 클로드를 '대화'를 넘어 **'일 맡기는 도구'** 로 쓰고 싶은 분

## 수업이 끝나면

> 4시간 동안 아래 세 가지를 직접 확인합니다.

- 클로드가 읽고 **요약·검색·정리**해주는 내 지식노트 볼트
- 다른 사람에게 공유할 수 있는, **인터넷에 떠 있는 내 사이트 URL**
- 혼자서도 이어갈 **30일 자립 로드맵**

## 진행 방식

| 경로 | 의미 | 목표 |
|------|------|------|
| **기본 진행** | 모두 같이 따라가는 경로 | Obsidian LLM 위키와 Vercel 라이브 URL 만들기 |
| **막힐 때** | 설치·계정·권한에서 막히는 경우 | 강사 데모나 대체 경로로 전체 흐름 확인 |
| **더 해볼 것** | 시간이 남거나 준비된 경우 | Supabase 기능, Cowork 폴더 작업, 추가 페이지 배포 |

## 준비물

| 구분 | 준비물 | 설명 | 비용 |
|------|------|------|------|
| **필수** | **GitHub · Vercel** | 사이트 배포와 자동 재배포에 필요. GitHub는 구글 계정으로 가입하면 Vercel 연결이 쉽다. | 무료 |
| **필수** | **옵시디언(Obsidian)** | 지식노트 '볼트'(.md 폴더). 변환 없이 클로드가 그대로 읽는다. | 무료 |
| **권장** | **클로드 유료 플랜 + 데스크탑 앱** | Claude Code·Cowork·파일 작업 실습에 필요. 안정성 위해 **Max 권장**(5x $100 / 20x $200·월). | 유료 |
| **권장** | **Node.js** | Claude Code npm 설치, 로컬 Next.js 실행, 사이트 빌드에 사용. Desktop Extensions 방식만 쓸 때는 별도 설치가 필요 없을 수 있다. | 무료 |
| **선택** | **Supabase** | 로그인/DB 기능 확장에 사용. 최소 배포 성공은 Vercel URL 확인까지로 잡는다. | 무료 |
| **선택** | **안티그래비티 IDE** | 구글 에이전트-우선 IDE. 무료 사용량 제한이 있어 Claude Code 보조로 사용한다. | 무료 |

## 커리큘럼 (240분)

> 시연 20% · **직접 실습 60%** · 결과 확인 20% — 각 교시 끝에 오늘 만든 것을 같이 확인합니다.

### 0교시. 세팅 & 워밍업 (20분)

- 오늘 만들 **결과물 2개** 미리보기
- 준비물 점검·설치 트러블슈팅, 계정 연결(**GitHub→Vercel→Supabase**)
- 안전 수칙 — '넣지 말 것' 레드리스트

> **확인할 것 — 세팅 완료**

### 1교시. LLM 위키 = 나만의 지식 노트 (60분)

- (개념 10') **RAG와 뭐가 다른가** — 매번 다시 찾는 RAG와 달리, LLM 위키는 **한 번 정리해 두고 소스가 늘 때마다 갱신**되는 누적·복리형 지식창고 (안드레아 카파시 [LLM Wiki 패턴](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f))
- (실습 30') 옵시디언 볼트 생성 → 노트 3~5개 + 백링크 `[[ ]]` → **클로드 데스크탑에 파일시스템 MCP 연결**(Extensions 원클릭 권장, JSON은 고급/대체 경로)
- (운영 20') 볼트 루트에 **`CLAUDE.md` 스키마** + **`index.md`·`log.md`** → **Ingest(새 소스 투입) · Query(질문 결과 저장) · Lint(건강검진)** 로 위키가 계속 갱신되게 만들기
- 클로드에게 내 볼트 **요약·검색·새 노트 생성·정리** 위임

> **확인할 것 — 계속 갱신되는 내 LLM 위키 (볼트 + CLAUDE.md + 정리 노트)**

### 2교시. Skill·MCP 개념 + Claude·Cowork 핸즈온 (60분)

- (개념 15') **Skill** = 일하는 방법 / **MCP·Connector** = 도구와 파일에 닿는 통로 / **Cowork** = 그 방법과 통로로 실제 일을 맡기는 화면
- (실습 35') 내장 Skill 체감(PDF·엑셀 자동 처리) → 공식 마켓플레이스 도구 확인(**Superpowers**, Playwright, Vercel, Supabase, GitHub) → 커넥터/MCP 원클릭 추가 → Cowork로 폴더 통째 작업해 실파일 산출
- (확인 10') Playwright CLI/Skills 또는 Playwright MCP로 배포 URL이 실제로 열리는지 점검하는 흐름 보기

> **확인할 것 — 내 커넥터·스킬로 만든 파일**

### 3교시. 실제 배포되는 사이트 만들기 ★실습 중심 (90분)

- (설계 15') 만들 사이트(프로필 + 즉흥 주제) · 스택 그림 **GitHub→Vercel→Supabase** · 무료 한도와 비상업 제한 고지
- (빌드 60') **Vercel 'Supabase Starter' Deploy** → **라이브 URL** 발급 → 내용·디자인 수정 → **git push → 자동 재배포** → 준비된 수강생은 Supabase 기능 확장

> **확인할 것 — 최소: 인터넷에 떠 있는 URL / 표준: 내 문구로 수정된 재배포 / 더 해볼 것: Supabase 기능 연결**

### 4교시. 마무리 & 자립 (10분)

- 오늘 만든 2개 결과 회고 + 라이브 URL 공유
- **30일 자립 로드맵** · 비용 정리 · Q&A

> **확인할 것 — 30일 자립 플랜**

## 사용 도구 — 가장 단순·안정한 경로

> 4시간 안에 실제 동작까지 보려면 경로를 단순하게 잡아야 합니다. 모두 **공식 문서로 확인한 경로**만 채택했습니다.

| 목표 | 채택 경로 (가장 쉬움) | 주의 |
|------|------|------|
| 지식노트(LLM 위키) | Obsidian 볼트 + **클로드 파일시스템 커넥터** | 볼트는 .md 폴더라 그대로 읽힘 |
| Skill·MCP 핸즈온 | 데스크탑 **Extensions 원클릭** + Cowork | Cowork는 유료·데스크탑 전용, JSON 설정은 고급/대체 경로 |
| 개발 워크플로우 Skill | Claude Code 공식 마켓플레이스의 **Superpowers** | 브레인스토밍, 디버깅, 코드 리뷰, TDD 흐름을 배우는 선택 확장. 서드파티이므로 설치 전 권한·출처 확인 |
| 브라우저 확인 | **Playwright CLI + Skills** 또는 Playwright MCP | 배포 URL 열기, 클릭, 스크린샷, 간단한 화면 점검. 수강생 필수라기보다 강사 시연·심화 경로 |
| 배포·DB·저장소 연결 | Vercel · Supabase · GitHub MCP/Plugin | 4시간 수업에서는 기본 배포가 우선. 시간이 남을 때 연결 |
| 배포되는 사이트 | Vercel **'Supabase Starter' Deploy** | 최소 성공은 라이브 URL. Vercel Hobby는 **비상업 전용**, Supabase Free는 **1주 비활성 시 일시정지** |
| 코딩 에이전트 | **클로드 코드(Max) 주력** + 안티그래비티 보조 | Claude Code npm 설치·로컬 빌드는 Node 필요, Antigravity는 선택 보조 |

> 비용·정책은 공식 문서 기준(2025년 말~2026년)이며 수업일 재확인이 원칙입니다.
