# Module 3: 에이전시에 에이전틱 팀 도입하기 (25분) ★메인

> 발표 목표: 에이전시 업무 흐름의 각 단계에서 AI가 어떻게 활용되는지 구체적으로 보여주고, 실제 운영 사례와 비용을 현실적으로 전달

---

## 에이전시 업무 흐름 × AI 자동화 매핑

에이전시의 전형적인 프로젝트 흐름과 AI 자동화 포인트:

| 단계 | 전통적 방식 | AI 에이전트 활용 | 자동화 수준 |
|------|-----------|----------------|-----------|
| **리서치** | 사람이 구글링 + 정리 | 리서치 에이전트가 자료 수집 → 구조화된 보고서 | 매우 높음 |
| **기획** | PM이 문서 작성 | AI가 리서치 기반으로 기획서 초안 작성 | 높음 |
| **디자인** | 디자이너 목업 | AI가 와이어프레임 + UI 초안 생성 | 보통 |
| **개발** | 개발자 코딩 | 코딩 에이전트가 구현 + 개발자가 리뷰 | 높음 |
| **QA** | QA 수동 테스트 | QA 에이전트가 테스트 코드 작성 + 실행 | 매우 높음 |
| **납품/운영** | 수동 배포 + 문서화 | 자동 배포 파이프라인 + 문서 자동 생성 | 높음 |

**핵심 메시지**: "모든 단계를 AI로 대체하는 게 아니다. **반복적이고 구조화된 작업을 AI에게 맡기고, 사람은 판단과 창의에 집중**하는 것이다."

---

## 실제 사례 — codemon.ai의 AI 팀

codemon 팀은 7마리의 AI 에이전트로 구성:

| 에이전트 | 역할 | 담당 업무 |
|---------|------|----------|
| 로디 | PM & 오케스트레이터 | 기획, 블로그 작성, 팀 조율 |
| 누비 | QA & Dev | 코드 리뷰, 테스트, 디버깅, 외주 대응 |
| 나래 | Content & Marketing | 리서치, 자료 수집, SEO |
| 베어 | Design & Frontend | UI/UX, 컴포넌트 설계 |
| 뉴비 | Research & Scout | 트렌드 분석, 경쟁사 조사 |
| 옥토 | Backend & Infra | 서버, API, 인프라 |
| 아울 | Data & Automation | 데이터 처리, 자동화 |

**에이전시 관점 포인트**: "1인 개발자가 7명의 AI 팀을 운영한다. 에이전시가 이 구조를 도입하면?"

---

## CLAUDE.md = AI 팀원의 온보딩 문서

새 팀원이 입사하면 온보딩 문서를 준다. AI 에이전트도 마찬가지.

**CLAUDE.md**: 프로젝트에 들어올 때 AI가 가장 먼저 읽는 파일

```markdown
# CLAUDE.md (예시)

## 프로젝트 개요
- 클라이언트: ○○ 에이전시
- 스택: Next.js 15 + Supabase + Vercel
- 배포: vercel build && vercel deploy --prebuilt

## 코딩 규칙
- 커밋 메시지: 한국어, 상세하게
- 테스트: 새 기능마다 반드시 작성
- 금지: rm -rf, force push to main

## 에이전트 역할
- code-reviewer: 모든 PR에 대해 보안/성능 리뷰
- research-scout: 기술 선택 시 대안 조사
```

**비개발자에게**: "이 문서 하나만 잘 쓰면, AI가 우리 팀의 규칙을 이해하고 따른다. 회사 컨벤션, 금지 사항, 선호하는 방식을 다 적어두면 된다."

---

## 훅(Hooks) = 자동 가드레일

에이전트가 자율적으로 일할 때 필요한 안전장치.

> "훅은 AI한테 거는 제한이 아니라, 업무 프로세스를 자동화하는 것이다."

| 훅 | 시점 | 에이전시 활용 예시 |
|----|------|-----------------|
| `PreToolUse` | 도구 실행 전 | 프로덕션 DB 직접 수정 차단 |
| `PostToolUse` | 도구 실행 후 | 코드 수정 시 자동 포맷팅 + 린트 |
| `SessionStart` | 세션 시작 | 프로젝트별 컨텍스트 자동 로드 |
| `Stop` | 작업 완료 | 슬랙/텔레그램으로 결과 알림 |

**실제 예시**: "개발 에이전트가 코드를 수정하면 → 자동으로 prettier가 돌고 → 테스트가 실행되고 → 실패하면 에이전트가 다시 수정 → 성공하면 슬랙에 알림"

---

## 스킬(Skills) = AI용 업무 매뉴얼

스킬은 재사용 가능한 워크플로우를 정의한 문서.

```markdown
# 리서치 스킬 (SKILL.md)
1. 주제 키워드로 웹 검색 (최소 5개 소스)
2. 각 소스에서 핵심 정보 추출
3. 결과를 구조화된 마크다운으로 정리
4. 출처 목록 포함
```

**에이전시 업무별 스킬 아이디어**:

| 스킬 | 용도 | 담당 에이전트 |
|------|------|-------------|
| 경쟁사 분석 | 클라이언트 경쟁사 5곳 조사 → 보고서 | 뉴비 (Research) |
| 기술 스택 추천 | 요구사항 기반 기술 스택 비교표 | 옥토 (Backend) |
| 코드 리뷰 | PR별 보안/성능/스타일 검토 | 누비 (QA) |
| 주간 보고서 | 프로젝트 진행 상황 자동 요약 | 로디 (PM) |
| 클라이언트 QA | 체크리스트 기반 기능 검증 | 아울 (Data) |

**비개발자에게**: "업무 매뉴얼을 문서로 작성하듯, AI한테 일 시키는 매뉴얼을 작성하는 것이다. 한 번 만들면 모든 프로젝트에서 재사용된다."

## 멀티 에이전트 파이프라인 — 실전 워크플로우

codemon 팀에서 블로그 글 하나를 만드는 파이프라인:

```
1. 로디(PM) → 주제 선정, 키워드 기획     [1분]
2. 나래(리서치) → 자료 수집, 보고서 작성   [5분]
3. 로디(블로그) → 블로그 글 작성          [2분]
4. 누비(QA) → 팩트체크, 검수, 발행        [5분]
────────────────────────────────────────
총 13분 → 발행 가능한 블로그 글 완성
```

**v1(36분) → v2(13분) → v3(9분)**: 파이프라인 최적화로 75% 시간 단축

**에이전시 프로젝트에 적용한다면**:

```
1. 뉴비(리서치) → 클라이언트 산업/경쟁사 조사   [10분]
2. 로디(기획) → 프로젝트 기획서 초안             [5분]
3. 베어(디자인) → 와이어프레임 초안               [15분]
4. 옥토(개발) → 프론트엔드 스캐폴딩              [20분]
5. 누비(QA) → 코드 리뷰 + 기본 테스트           [10분]
────────────────────────────────────────────
총 ~1시간 → 클라이언트 프레젠테이션용 프로토타입
```

## 2봇 협업 — 현실적인 시작점

모든 에이전시가 7마리 에이전트부터 시작할 필요 없다. **2대부터 시작하자.**

codemon의 시작: 맥미니 1대(로디) + 맥북 1대(누비)

**협업 구조**:
- iCloud 공유 폴더로 비동기 통신
- GitHub으로 스킬 공유
- 역할 비대칭: 로디=상주 서버 관리자, 누비=이동형 보조

**핵심 설계 원칙**:

| 원칙 | 이유 |
|------|------|
| 비동기 > 실시간 | 상시 연결 불필요한 환경에서 과잉 설계 방지 |
| 파일 > 프로토콜 | 디버깅 쉽고, 사람도 읽을 수 있고, 도구 의존성 없음 |
| 역할 분리 | 대칭 구조는 충돌을 유발. 비대칭이 자연스럽다 |

## 비용 현실 — 솔직하게

### 모델별 가격 (2026년 2월)

| 모델 | Input ($/1M tokens) | Output ($/1M tokens) | 용도 |
|------|---------------------|----------------------|------|
| Opus 4.6 | $15 | $75 | 복잡한 추론, 아키텍처 |
| Sonnet 4.6 | $3 | $15 | 일상 개발 (가성비 최고) |
| Haiku 4.5 | $1 | $5 | 단순 검색, 분류 |

### 에이전시 규모별 비용 시뮬레이션

| 규모 | 구성 | 월 비용 | 비고 |
|------|------|--------|------|
| **1인 (시작)** | Claude Pro ($20) | ~$20/월 | 서브에이전트 가끔 사용 |
| **소규모 팀** | Max 5x ($100) | ~$100/월 | Agent Teams 일상 사용 |
| **적극 활용** | Max 20x ($200) + API | $200-400/월 | 대규모 병렬 작업 |
| **풀 파이프라인** | API 직접 사용 | $500+/월 | 자동화 파이프라인 24시간 운영 |

> 개발자 1명 평균: 하루 ~$6, 90%가 $12/일 이하

### 비용 최적화 전략

1. **모델 분리**: 복잡한 판단은 Opus, 일상 코딩은 Sonnet, 단순 작업은 Haiku
2. **프롬프트 캐싱**: 반복 요청 최대 90% 절감
3. **Agent Teams 주의**: 일반 세션 대비 ~7배 토큰 → 필요한 작업에만 사용

**비개발자에게**: "신입 개발자 한 명 인건비의 1/10~1/20 수준으로 AI 팀을 운영할 수 있다. 대신 '잘 시키는 법'을 아는 사람이 필요하다."

## 에이전시에서 바로 적용 가능한 시나리오

### 시나리오 1: 프로젝트 킥오프 자동화
"새 클라이언트 ○○ 쇼핑몰 리뉴얼 프로젝트" → 리서치 에이전트가 경쟁사 5곳 분석 → 기획 에이전트가 RFP 초안 → PM이 리뷰만

### 시나리오 2: 코드 리뷰 자동화
개발자가 PR을 올리면 → QA 에이전트가 보안/성능/스타일 리뷰 → 결과를 슬랙에 알림 → 개발자는 AI 리뷰를 참고해서 수정

### 시나리오 3: 주간 보고 자동화
금요일 오후 → 에이전트가 GitHub 커밋, 슬랙 대화, 프로젝트 보드를 분석 → 주간 보고서 자동 생성 → PM이 검토 후 클라이언트에게 전달
