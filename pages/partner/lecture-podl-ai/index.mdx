# AI로 일하는 방법

## 개요

| 항목 | 내용 |
|------|------|
| 제목 | AI로 일하는 방법 — 포들 실무 중심 AX 가이드 |
| 형식 | 강의 + 라이브 데모 + 핸즈온 (3시간) |
| 대상 | 라스트스프링(PO:DL) 전 직무 실무자 17명 |
| 강사 | 코드몬 (codemon.ai) |

## 이런 분에게 추천합니다

- AI를 써보고 싶은데, **내 직무에서 어떻게 쓰는지** 모르겠는 실무자
- 반복 업무에 시간을 뺏겨서 **기획·전략에 집중할 여유가 없는** 팀
- 데이터는 있는데 **취합·분석·공유가 안 돼서** 기회를 놓치고 있는 조직

## 핵심 성과

> AI를 잘 쓰는 17명이 AI를 모르는 170명을 이긴다

- 인플루언서 시딩 이메일 20통을 **한번에 동시 생성**
- 흩어진 3채널 매출 데이터를 **일일 자동 리포트 + 차트**로 전환
- SNS 히트 콘텐츠를 **자동 감지** → 전사 공유 → 후속 전략 즉시 수립
- 상세페이지 + SNS + 해시태그 + 틱톡 **4종 콘텐츠 동시 생성**
- 마케팅 카피 **5개국 문화 맞춤 현지화**
- 패키지 라벨 **4개국 규제 자동 반영**
- 반품 사유 패턴 분석 + **CS 응대 메시지 자동 생성**

## 커리큘럼 (3시간)

> 총 180분 = 강의 145분 + Q&A 20분 + 여유/쉬는 시간 15분 | 데모 7개 + AI 대시보드 + Stitch 핸즈온

### Part 1. AI 기본 교양 — "AI, 대체 뭔데?" (25분)

데모를 보기 전에, AI가 뭔지 / 어떤 단어를 알아야 하는지 / 어떻게 동작하는지 / 왜 지금 난리인지를 정리합니다.

#### 1-1. 지금 세상에서 일어나고 있는 일 (3분)

> 2026년 2월, Claude Cowork 출시 48시간 만에 소프트웨어 주식 **413조원 증발**.
> 시장이 "이건 진짜 패러다임 전환이다"라고 반응한 것입니다.

- **Rakuten**: 1,250만 줄 코드를 AI가 자율 관리 — 인간 코드 기여 **0%**
- **Zapier**: 조직 전체 89% AI 채택, 800+ 에이전트 내부 운영
- **Lovable**: 비개발자 63%가 앱을 직접 만드는 시대 — 12개월 만에 매출 5천억
- **SK AX**: "성과 기준이 '얼마나 했느냐'에서 **'AI를 얼마나 잘 지휘했느냐'**로 이동"
- **한국 기업 85%**가 생성형 AI 도입 — 그런데 60%는 아직 실험 단계

> 세상은 이미 바뀌고 있습니다. 문제는 "할 것인가 말 것인가"가 아니라 **"어떻게 잘 할 것인가"**입니다.

#### 1-2. AI 용어 사전 — 이것만 알면 됩니다 (5분)

> 오늘 강의에서 나올 핵심 단어 6개. 이것만 알고 가세요.

| 용어 | 비유 | 한 줄 설명 |
|------|------|----------|
| **프롬프트** | 주문서 | AI에게 하는 지시. "이거 해줘"가 프롬프트 |
| **모델** | 요리사의 실력 | Claude, GPT, Gemini는 각각 다른 요리사. 실력(성능)이 다름 |
| **토큰** | 글자 수 = 비용 | AI가 읽고 쓰는 단위. 많이 쓰면 비용 ↑ |
| **컨텍스트** | 배경 서류 | AI에게 주는 참고 자료. 많이 줄수록 결과가 좋아짐 |
| **에이전트** | 자율 팀원 | 지시하면 알아서 찾아보고, 실행하고, 보고하는 AI |
| **MCP** | USB 포트 | AI를 외부 도구(슬랙, 구글시트 등)에 꽂는 규격 |

> 이 6개 단어가 오늘 계속 나옵니다. 모르면 물어보세요!

#### 1-3. AI는 어떻게 동작하는가 (7분)

**핵심 원리: "다음 단어 예측"**

AI는 마법이 아닙니다. "앞에 이 단어가 왔으니, 다음에 올 확률이 가장 높은 단어"를 선택하는 것입니다.

> 예: "보들보들한 피부를 위한 ___" → AI가 확률적으로 "부드러움"을 선택

**왜 환각(hallucination)이 생기는가?**

모르는 것도 확률적으로 "그럴듯하게" 만들어냅니다. "보들의 매출은 ___"이라고 물으면 — 우리 매출을 모르지만 그럴듯한 숫자를 만들어냄. **이게 AI의 한계이자, 컨텍스트가 중요한 이유입니다.**

**왜 컨텍스트가 게임 체인저인가 — 예시로 이해하기**

**신입사원 vs 3년차:**
- 신입에게 "시딩 이메일 써줘" → 브랜드 톤도 모르고, 인플루언서도 모름 → 아무 회사나 보낼 수 있는 이메일
- 3년차에게 "시딩 이메일 써줘" → 보들 톤 알고, 팀장 스타일 알고, 인플루언서 취향도 앎 → "이 브랜드 잘 아네" 하는 이메일
- **차이 = 머릿속에 있는 맥락(컨텍스트)의 양. AI도 똑같습니다.**

**실제 Before/After:**

| | Before (프롬프트만) | After (컨텍스트 추가) |
|---|---|---|
| 입력 | "밤투폼 시딩 이메일 써줘" | "밤투폼 시딩 이메일 써줘" + **브랜드 가이드** + **인플루언서 프로필** + **제품 성분표** |
| 결과 | "안녕하세요, 저희 제품을 소개합니다..." | "서연님, 최근 민감피부 루틴 영상 잘 봤어요! 시어버터+판테놀 조합이..." |

> **똑같은 AI, 똑같은 프롬프트인데 결과가 완전히 다릅니다. 차이는 오직 컨텍스트.**

**포들에서 컨텍스트가 되는 파일들:**
- `브랜드_가이드.md` — 보들의 톤, 슬로건, 금지 표현
- `제품_성분표.csv` — 5개 제품의 성분, USP, 타겟
- `인플루언서_DB.csv` — 20명의 채널, 팔로워, 관심사

> "이 세 파일을 AI에게 주면, AI는 포들을 3년 다닌 직원처럼 일합니다."
> **오늘 데모에서 실제로 보여드리겠습니다.**

#### 1-4. AI 도구 지도 — 뭐가 있는지 알아야 고른다 (5분)

**대표 AI 모델 — 요리사가 여러 명입니다**

| AI | 만든 곳 | 강점 | 우리가 쓸 곳 |
|----|--------|------|-----------|
| **Claude** | Anthropic | 코딩, 에이전트, 긴 문서 분석 | 오늘 데모 전체 |
| **ChatGPT** | OpenAI | 가장 대중적, 플러그인 생태계 | 일반 질의응답 |
| **Gemini** | Google | 검색 연동, 2M 토큰 컨텍스트 | 리서치, 문서 분석 |

**바이브코딩 도구 — 비개발자도 앱을 만든다**

| 도구 | 특징 |
|------|------|
| **Lovable** | 자연어로 설명하면 앱 완성 (사용자 63% 비개발자) |
| **Google Stitch** | AI로 디자인 생성 (무료 350회/월) — 오늘 핸즈온 |
| **v0** (Vercel) | React 컴포넌트 자동 생성 |

**자동화 도구 — AI 결과를 업무에 연결**

| 도구 | 역할 |
|------|------|
| **Make / Zapier** | AI 결과 → 슬랙/이메일/시트 자동 연결 |
| **MCP** | AI를 쿠팡/인스타/Gmail에 직접 연결하는 표준 |

> **오늘 주로 쓸 것**: Claude(데모 1~5,7) + Stitch(데모 6 핸즈온) + MCP(대시보드 비전)

#### 1-5. "다들 AI 쓴다는데, 왜 효과가 없죠?" — 3단계로 해결 (5분)

**왜 안 되는 걸까요? 원인은 3가지입니다.**

| 원인 | 여러분이 겪는 현상 | 오늘의 해결법 |
|------|-----------------|-------------|
| **AI에게 줄 재료가 없다** | "시딩 이메일 써줘" → 결국 다시 씀 | **컨텍스트** = 배경 서류를 항상 함께 줌 |
| **일회성으로 끝난다** | 다음 날 같은 프롬프트를 못 찾음 | **스킬** = 반복 프롬프트를 저장해서 재사용 |
| **업무 흐름에 안 붙는다** | 슬랙 공유하려면 복붙 → 안 함 | **자동화** = 결과를 바로 연결 |

**AI 활용 3단계: 프롬프트 → 컨텍스트 → 하네스**

| 단계 | 비유 | 포들 적용 |
|------|------|---------|
| **① 프롬프트** | 인턴에게 구두 지시 | "시딩 이메일 써줘" — 매번 처음부터 설명 |
| **② 컨텍스트** ★ | 인턴에게 매뉴얼 전달 | 브랜드 가이드 + 제품 정보를 파일로 항상 전달 |
| **③ 하네스** | 부서 시스템 구축 | 매출 리포트 자동 생성 → 슬랙 발송 (다음 단계) |

> **오늘의 목표**: ②번 **컨텍스트** 단계를 확실히 익히는 것.
> ③번은 다음 단계. 하지만 ②번은 **내일부터 바로 효과**가 나옵니다.

### Part 2. "포들에 적용하면?" — 우리 이야기 (10분)

AI 기본을 익혔으니, 이제 우리 회사 이야기입니다.

#### 2-1. 포들의 하루 — 7개 팀의 병목 (4분)

> "이 중에 해당되는 거 있으면 손 들어보세요"

| 팀 | 지금 겪고 있는 일 |
|----|-----------------|
| 마케팅 (박서연) | 인플루언서 20명한테 이메일 쓰면 **하루 종일** |
| MD (정우진) | 대표님이 "오늘 매출 얼마야?" → 담당자 자리로 **걸어감** |
| 콘텐츠 (이하은) | 상세페이지 하나에 **반나절**, SNS까지 하면 하루 |
| 마케팅 (박서연) | 릴스 300만뷰 터졌는데 **전사 아무도 모름** |
| 디자인 (최민지) | 미국용 라벨 만들려면 **FDA 규격 찾는 것부터** |
| 미국팀 (Sarah) | 번역 외주 맡기면 **"보들보들함"이 사라짐** |
| 물류 (강태호) | 반품 사유 하나하나 읽고 **엑셀에 집계** |

그래서 **AI를 활용해 효율적으로 일하는 방법**이 필요합니다.

#### 2-2. 이걸 AI가 해준다면? (3분)

> "오늘 이 자리에서, 여러분 회사 데이터로 직접 보여드리겠습니다"

- 시딩 이메일 20통 → **한번에 동시 생성**
- 3채널 매출 → **자동 리포트 + 차트**
- SNS 30개 게시물 → **히트 패턴 자동 분석**
- 제품 1개 → **상세페이지 + 캐러셀 + 해시태그 + 틱톡 동시**
- 마케팅 카피 → **5개국 문화 맞춤 현지화**
- 패키지 라벨 → **4개국 규제 자동 반영**
- 반품 200건 → **사유 패턴 + CS 응대 메시지**

이건 마법이 아니라 **도구**입니다. 오늘 7개를 전부 라이브로 보여드립니다.

#### 2-3. 데모를 만든 이유 + 미리보기 (3분)

> 오늘 보여드릴 7개 데모는 랜덤이 아닙니다.
> **여러분 7개 팀의 실제 병목**에서 출발했습니다.

| 데모 | 팀 | 핵심 |
|------|----|------|
| 시딩 이메일 | 마케팅 | 컨텍스트 (인플루언서 DB + 톤 가이드) |
| 매출 리포트 | MD+회계 | 컨텍스트 (채널 데이터 통합) |
| 소재 트래킹 | 마케팅 | 컨텍스트 (SNS 성과 데이터) |
| 콘텐츠 생성 | 콘텐츠 | 컨텍스트 (브랜드 톤 + 제품 정보) |
| 글로벌 현지화 | 미국팀 | 컨텍스트 (문화권 지식) |
| 패키지 라벨 | 디자인 | 컨텍스트 (규제 데이터) |
| 반품 분석 | 물류 | 컨텍스트 (반품 데이터) |

> 전부 **컨텍스트**가 핵심입니다. AI에게 "우리 회사 데이터"를 주는 순간, 결과가 달라집니다.
> **7개 팀 전체**를 커버합니다 — 여러분 팀의 데모가 반드시 있습니다. 지금부터 시작합니다.

### Part 3. "포들 실무에 AI 적용하기" — 라이브 데모 7선 (65분)

> 각 데모는 **Claude 위주**로 진행합니다. 별도 개발 없이, 실무자가 직접 따라 할 수 있는 수준입니다.
> 각 데모 페이지에 📖 설명 버튼이 있어 동작 원리를 다이어그램으로 확인할 수 있습니다.

#### Demo 1. 인플루언서 시딩 이메일 자동화 (10분) — 마케팅팀

**지금의 문제**: 인플루언서에게 하나하나 이메일 작성·답장 → 업무의 80%를 차지

- 인플루언서 리스트(이름, 팔로워, 관심사, 채널) + 포들 제품 정보 입력
- Claude가 각 인플루언서별 **개인화 시딩 이메일 벌크 생성**
- 톤/매너 자동 조정 (10가지 프리셋), 제품-인플루언서 매칭
- **후속 자동화**: Make/Zapier 연동으로 자동 발송 파이프라인 구축 가능

> [📌 데모 바로가기](/partner/lecture-podl-ai/demo/seeding)

#### Demo 2. 매출 데이터 일일 리포트 자동 생성 (10분) — MD팀 + 회계팀

**지금의 문제**: 대표가 각 담당자 자리에 찾아가서 오늘 매출이 얼마인지 물어봐야 한다

- 3채널(쿠팡/자사몰/올리브영) 1,000건 주문 데이터 입력
- Claude가 **통합 분석 + 채널별 비교 + 이상치 감지 + SVG 차트 5종**
- 경영진용 리포트 자동 생성 → 새창에서 차트 포함 리포트 확인

> [📌 데모 바로가기](/partner/lecture-podl-ai/demo/report)

#### Demo 3. SNS 소재 성과 트래킹 + 슬랙 알림 (10분) — 마케팅팀

**지금의 문제**: 릴스 300만뷰가 터져도 전사 모두가 모르고 있다

- 인스타/틱톡 30개 게시물 성과 데이터 입력
- AI가 **히트 패턴 분석** + 플랫폼별/유형별 필터 + 정렬 테이블

### Part 4. "AI로 디자인까지" — Stitch 핸즈온 (15분)

> Demo 6의 확장. Claude Code에서 Google Stitch 2.0으로 **패키지 라벨 디자인을 직접 생성**합니다.

#### 핸즈온 시연: Claude Code + Stitch MCP

1. **프로젝트 생성** — "보들 패키지 라벨 만들어줘" 한 줄로 Stitch 프로젝트 생성
2. **브랜드 시스템 등록** — DESIGN.md(색상/폰트/톤)를 Stitch에 자동 업로드
3. **한국어 라벨 생성** — 밤투폼 정면+후면 라벨 디자인 자동 생성
4. **다국어 변환** — 같은 디자인을 미국 FDA / 아랍어 할랄 / EU 프랑스어로 변환
5. **브랜드 톤 일괄 적용** — 3개국 라벨에 보들 디자인 시스템 통일

**포인트**: 코딩 없이, 프롬프트만으로 디자인 → 다국어 변환 → 브랜드 통일까지.
최민지 팀장이 내일부터 바로 쓸 수 있습니다.

**디자인팀만이 아닙니다**: 콘텐츠팀(SNS 배너/크리에이티브), 마케팅팀(캠페인 비주얼), 미국팀(해외 상세페이지 시안) — 모든 팀이 Stitch로 디자인 초안을 직접 만들 수 있습니다.

> **Google Stitch**: 무료 350회/월, Gmail만 있으면 사용 가능

#### 컨셉: 포들 AI 컨트롤 센터

| 패널 | 데이터 소스 | 역할 | 데모 |
