# AI로 일하는 방법

## 개요

| 항목 | 내용 |
|------|------|
| 제목 | AI로 일하는 방법 — 포들 실무 중심 AX 가이드 |
| 형식 | 강의 + 라이브 데모 (2시간) |
| 대상 | 라스트스프링(PO:DL) 전 직무 실무자 |

## 이런 분에게 추천합니다

- AI를 써보고 싶은데, **내 직무에서 어떻게 쓰는지** 모르겠는 실무자
- 반복 업무에 시간을 뺏겨서 **기획·전략에 집중할 여유가 없는** 팀
- 데이터는 있는데 **취합·분석·공유가 안 돼서** 기회를 놓치고 있는 조직

## 핵심 성과

> AI를 잘 쓰는 17명이 AI를 모르는 170명을 이긴다

- 인플루언서 시딩 이메일 작성 시간 **80% → 15%** 단축
- 흩어진 매출 데이터를 **일일 자동 리포트**로 전환, 대표가 슬랙에서 바로 확인
- SNS 히트 콘텐츠를 **실시간 감지** → 전사 공유 → 후속 전략 즉시 수립

## 커리큘럼 (2시간)

> 총 120분 = 강의 100분 + Q&A 15분 + 여유 5분 | 데모 5개 + AI 대시보드 비전 포함

### Part 1. "AI, 어떻게 쓰는 건가?" (15분)

AI 활용의 3단계와 실무에서 쓰는 핵심 개념을 정리합니다.

#### 1-1. AI 활용 3단계: 프롬프트 → 컨텍스트 → 하네스

- **프롬프트**: 잘 시키면 잘 하는 인턴 — 대부분의 실무자는 여기서 시작합니다
- **컨텍스트**: 파일로 유지하는 팀의 지식 — **전사 도입의 핵심**
  - 예: "포들 브랜드 가이드", "CS 응대 원칙", "제품 성분표"를 AI에게 항상 주면 품질이 급격히 올라갑니다
  - **퍼시스턴트 메모리**: 매번 설명하지 않아도 AI가 우리 회사를 기억하게 하려면, 핵심 문서를 **파일로 유지**하는 것이 가장 중요합니다
- **하네스**: 자동화 파이프라인 — 포들은 아직 여기까진 불필요하지만, 알아두면 다음 단계로 갈 준비가 됩니다

#### 1-2. 스킬과 MCP: Claude를 단순 챗봇 그 이상으로

- **스킬**: 반복 작업을 패턴화해서 한 번에 실행 (예: "시딩 이메일 만들어줘" → 전체 워크플로우 자동 실행)
- **MCP(Model Context Protocol)**: 외부 도구·데이터와 Claude를 연결하는 표준
  - 구글시트, 슬랙, 쿠팡 데이터 등을 Claude가 직접 읽고 쓸 수 있게 됩니다
  - **커스텀 MCP**: 포들 맞춤 MCP가 필요하면 codemon.ai가 개발을 지원합니다
  - 예: 쿠팡 매출 데이터 자동 수집 MCP, 인스타그램 성과 수집 MCP, Gmail 자동 발송 MCP

#### 1-3. 오늘 가져갈 것

- 포들 실제 데이터로 시연하는 **AX 데모 5개**
- 5개 데모를 하나로 묶는 **"포들 AI 대시보드"** 비전
- 내일부터 시작할 수 있는 구체적 로드맵

### Part 2. "포들 실무에 AI 적용하기" — 라이브 데모 5선 (55분)

> 각 데모는 **Claude 위주**로 진행합니다. 별도 개발 없이, 실무자가 직접 따라 할 수 있는 수준입니다.

#### Demo 1. 인플루언서 시딩 이메일 자동화 (12분)

**지금의 문제**: 인플루언서에게 하나하나 이메일 작성·답장 → 업무의 80%를 차지

- 인플루언서 리스트(이름, 팔로워, 관심사, 채널) + 포들 제품 정보 입력
- Claude가 각 인플루언서별 **개인화 시딩 이메일 벌크 생성**
- 톤/매너 자동 조정 (캐주얼↔포멀), 제품-인플루언서 매칭
- **후속 자동화**: Make/Zapier 연동으로 자동 발송 파이프라인 구축 가능
- **커스텀 MCP 예시**: Gmail API 연동 → Claude에서 직접 이메일 발송

> **데모 1 — 시딩 이메일 벌크 생성**: 인플루언서 10명 리스트를 입력하면 Claude가 각각의 프로필에 맞는 개인화 시딩 이메일을 생성하는 과정을 시연합니다

#### Demo 2. 매출 데이터 일일 리포트 자동 생성 (10분)

**지금의 문제**: 대표가 각 담당자 자리에 찾아가서 오늘 매출이 얼마인지 물어봐야 한다

- 여러 채널(쿠팡/자사몰/올리브영) 매출 CSV 입력
- Claude가 **통합 분석 + 전일 대비 + 채널별 비교 + 이상치 감지**
- 일일 리포트 자동 생성 → 슬랙 메시지 포맷으로 출력
- **커스텀 MCP 예시**: 쿠팡 파트너스 API 연동 → 매출 데이터 자동 수집

> **데모 2 — 일일 매출 리포트**: 3개 채널 매출 데이터를 입력하면 Claude가 통합 분석 리포트를 생성하고 슬랙 메시지 포맷으로 출력합니다

#### Demo 3. SNS 소재 성과 트래킹 + 슬랙 알림 (10분)

**지금의 문제**: 릴스 300만뷰가 터져도 전사 모두가 모르고 있다

- 인스타/틱톡 콘텐츠 성과 데이터(조회수, 좋아요, 댓글, 공유) 입력
- AI가 **성과 패턴 분석** + 임계치 초과 콘텐츠 하이라이트
- **"어떤 소재가 왜 잘 됐는지"** 자동 분석 리포트 생성
- 슬랙 알림 자동화 시나리오: "밤투폼 릴스 10만뷰 돌파!" → 전사 채널
- **커스텀 MCP 예시**: 인스타그램 Graph API 연동 → 실시간 성과 수집

> **데모 3 — 소재 성과 분석**: 최근 30개 게시물 성과 데이터로 AI가 히트 패턴을 분석하고 다음 콘텐츠 전략을 제안합니다

#### Demo 4. 상품 콘텐츠 자동 생성 (10분)

**지금의 문제**: 디자인팀·콘텐츠팀이 기획·제작에 시간을 과도하게 쓰고 있다

- 밤투폼 클렌저 제품 정보(성분, USP, 타겟) 입력
- **상세페이지 카피 + 인스타 캐러셀 문구 5종 + 해시태그** 동시 생성
- 컨텍스트 활용: 포들 브랜드 가이드 파일을 함께 넣으면 톤이 일관됩니다
- AI가 초안을 만들고 → 디자인팀은 **편집·검수에만 집중**

> **데모 4 — 상품 콘텐츠 생성**: 밤투폼 제품 정보를 입력하면 상세페이지 카피 + SNS 캐러셀 5종 + 해시태그가 동시에 생성됩니다

#### Demo 5. 글로벌 마케팅 카피 현지화 (8분)

**지금의 문제**: 미국팀 확장 중이지만, 번역 외주의 뉘앙스 손실이 크다

- 한국어 마케팅 카피 → 영어/아랍어 3개국 동시 현지화
- **단순 번역 vs 문화권 맞춤 로컬라이제이션** 차이를 실시간 비교
- 아마존 US / 중동 시장용 제품 설명 생성 사례

> **데모 5 — 글로벌 현지화**: 밤투폼 한국어 설명을 미국·중동 시장용 마케팅 카피로 현지화합니다. 단순 번역이 아닌, 문화권별 맥락을 반영한 카피라이팅입니다

### Part 3. "포들 AI 대시보드" — 5개를 하나로 (15분)

> 5개 데모가 각각의 솔루션이 아니라, **하나의 대시보드로 통합**되는 비전을 보여드립니다.

#### 컨셉: 포들 AI 컨트롤 센터

| 패널 | 데이터 소스 | 역할 |
|------|-----------|------|
| 매출 리포트 | 쿠팡/자사몰/올리브영 | 3채널 통합 일일 리포트 자동 생성 |
| 소재 트래킹 | 인스타/틱톡 | 히트 콘텐츠 실시간 감지 + 슬랙 알림 |
| 시딩 파이프라인 | Gmail/인플루언서 DB | 시딩 진행률·응답률 대시보드 |
| 글로벌 현황 | 아마존/해외몰 | EN/AR 현지화 진행률 + 해외 매출 |
| 콘텐츠 파이프라인 | 내부 워크플로우 | 기획 → AI 초안 → 편집 → 발행 → 성과 |

**실제 사례 — Paperclip**: codemon.ai가 운영 중인 AI 에이전트 회사의 컨트롤 플레인. AI 에이전트들이 자율적으로 리서치, 글 작성, 이미지 생성, 배포까지 수행하고 그 현황이 하나의 대시보드에 모입니다.

**포들 맞춤 버전**: 커스텀 MCP로 각 채널(쿠팡/인스타/Gmail 등)을 연동하면, Claude가 데이터를 자동 수집·분석하고 대시보드에 업데이트합니다.

> "프롬프트 잘 쓰기가 아니라 **시스템 잘 만들기** — 컨텍스트 파일 + 스킬 + MCP → 그리고 결국 이런 대시보드"

**codemon.ai 개발 지원**: 커스텀 MCP 개발이 필요한 부분은 codemon.ai가 직접 개발을 지원합니다. 포들의 기존 시스템(쿠팡 파트너스, 인스타 Graph API, Gmail 등)과 Claude를 연결하는 MCP를 제작하고, 대시보드 구축까지 함께 진행할 수 있습니다.

### Part 4. 내일부터 시작하기 + Q&A (15분)

#### 1주차: Quick Win

- Claude로 **가장 반복적인 업무 1개** 대체 (추천: 시딩 이메일)
- 프롬프트 템플릿 3개 만들어 팀 공유
- 포들 브랜드 가이드 파일 만들어서 **컨텍스트**로 활용 시작

#### 2~4주차: 팀 확산

- 직무별 AI 활용 베스트 프랙티스 공유 세션
- 프롬프트 라이브러리 구축 (노션/구글드라이브)
- AI 결과물 품질 체크리스트 만들기

#### 1~3개월: 자동화 파이프라인

- 커스텀 MCP 개발 (매출 데이터 수집, SNS 성과 트래킹, 이메일 자동화)
- 포들 AI 대시보드 v1 구축
- AI 활용 KPI 측정: 시딩 이메일 시간 80%→15%, 리포트 자동화, 히트 콘텐츠 감지율

#### 추천 도구

| 용도 | 도구 | 활용 시나리오 |
|------|------|-------------|
| 콘텐츠 생성 | Claude / ChatGPT | 카피라이팅, 상세페이지, SNS 포스팅, 시딩 이메일 |
| 이미지 생성 | Midjourney / DALL-E | 제품 이미지 배경, SNS 크리에이티브 |
