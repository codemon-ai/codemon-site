# Claude Code Agent Teams 설정 가이드 (Reddit 번역)

> 원문: [How to Set Up Claude Code Agent Teams](https://www.reddit.com/r/ClaudeCode/comments/1qz8tyy/)

---

Claude Code가 Agent Teams를 출시했는데, 이건 단순히 "서브 에이전트에 예쁜 이름 붙인 것"이 아니다. 3~5개의 독립적인 Claude Code 인스턴스가 **같은 프로젝트에서 실제로 협업**하고, 컨텍스트를 공유하며, 메시지를 주고받고, 공유 태스크 시스템을 통해 조율할 수 있는 완전히 다른 실행 모델이다.

내부적으로 어떻게 작동하는지 이해하려고 로그와 파일시스템 변경사항을 파헤치느라 꽤 오래 걸렸다. 알고 보니 기존 task 도구와 상당히 다르고, Agent Teams가 일반 서브 에이전트보다 확실히 나은 특정 상황들이 있다.

## 핵심 차이점

**기존 서브 에이전트 모델:** 메인 에이전트가 task 도구 호출 → 서브 에이전트 생성 → 격리된 상태로 작업 → 세션 종료 → 요약만 돌아옴

**새로운 Agent Teams 모델:** 공유 태스크 리스트, 에이전트 간 다이렉트 메시징, 명시적 라이프사이클 제어(시작, 종료). 에이전트들이 사일로에서 일하는 대신 실시간으로 조율하고, 토론하고, 서로 업데이트할 수 있다.

## 내부 작동 원리

Agent Teams는 내부적으로 5개의 새로운 도구를 사용한다:

1. **TeamCreate** – 팀 구조 설정 (`.claude/teams/` 아래에 폴더 생성)

2. **TaskCreate** – JSON 파일로 태스크 추가, 상태 추적, 의존성, 소유권 관리 (기존 Task 도구와 다름, 할 일 생성 전용)

3. **Task 도구 (업그레이드됨)** – 여전히 에이전트를 생성하지만, 이제 `name`과 `team_name` 파라미터로 단순 서브 에이전트 모드 대신 팀 모드 활성화 가능

4. **taskUpdate** – 에이전트들이 태스크를 할당받고, 상태 업데이트하고, 완료 표시하는 데 사용

5. **sendMessage** – 진짜 핵심 기능. 다이렉트 메시지(에이전트 → 에이전트)와 브로드캐스트(에이전트 → 모든 팀원) 지원. 메시지는 `.claude/teams/<team_id>/inbox/`에 저장되고, 각 에이전트의 대화 기록에 `<teammate-message teammate_id="...">`로 주입됨.

팀 리더가 `shutdown_request`를 보내면, 팀원들이 `shutdown_response`로 확인하고, 세션이 깔끔하게 종료된다.

## Agent Teams가 진짜 유용한 경우

지금까지 최고의 사용 사례: **여러 가설을 가진 깊은 디버깅**

공식 문서 예시: 사용자들이 앱이 연결 유지 대신 메시지 하나 후 종료된다고 보고함. 5명의 에이전트 팀원을 생성해서 각각 다른 이론을 조사. 서로 대화하고, 과학적 토론처럼 상대방 아이디어를 반박하려 시도하고, 합의가 나온 내용으로 findings 문서 업데이트.

이런 협업적, 다각도 조사는 요약만 보고하는 격리된 서브 에이전트로는 훨씬 어렵다.

## Agent Teams 설정 방법

### Step 1: Claude Code 최신 버전으로 업데이트

### Step 2: 실험적 플래그 활성화

설정 파일 열기:
```bash
code ~/.claude/settings.json
```

글로벌 설정에 추가:
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

파일 저장하고 터미널 재시작.

### Step 3: 새 Claude Code 세션 시작

Agent Teams는 프롬프트에서 명시적으로 팀 생성을 요청할 때 활성화된다. 예시:

> "코드베이스 전체에서 TODO 주석을 추적하는 CLI 도구를 설계 중이야. 에이전트 팀을 만들어서 다양한 각도로 탐색해줘: 한 팀원은 UX, 한 팀원은 기술 아키텍처, 한 팀원은 악마의 대변인 역할."

### 프로 팁: tmux나 iTerm2로 최고의 경험

Agent Teams는 모든 에이전트가 병렬로 작업하는 걸 볼 수 있을 때 빛난다.

**iTerm2 (macOS) 설정:**
- iTerm2 설치
- Settings → General → Magic 이동
- Python API 활성화
- iTerm2 재시작
- Claude Code 실행: `claude --teammate-mode tmux`

이렇게 하면 팀 리더용 패널 하나와 각 에이전트 팀원용 별도 패널이 열린다. 아무 패널이나 클릭해서 에이전트가 실시간으로 뭘 하는지 보고, 개별 에이전트에게 다이렉트 메시지도 보낼 수 있다.
