## 설문 문항 4개 추가 (learnings / follow_along / would_help / improvements)

모든 `/survey/[lectureId]` 페이지에 공통 4문항을 추가. 전 강의 공유 `SurveyForm` 컴포넌트 한 곳만 바꿔도 반영됨.

### 추가된 문항 (모두 선택 입력, textarea)
- 본인이 배운 것들 → `learnings`
- 어디까지 따라 갔는지 → `follow_along`
- 어떤 것이 있으면 도움이 될지 → `would_help`
- 개선이 되었으면 하는 부분 → `improvements`

기존 `gains`(강의에서 얻은 점) / `questions`(더 궁금한 점)는 하위 호환 위해 유지.

### 수정 파일

### Supabase 마이그레이션 (수동 실행 필요)

Supabase SQL Editor에서 실행:

```sql
ALTER TABLE survey_responses
  ADD COLUMN IF NOT EXISTS learnings text,
  ADD COLUMN IF NOT EXISTS follow_along text,
  ADD COLUMN IF NOT EXISTS would_help text,
  ADD COLUMN IF NOT EXISTS improvements text;
```

**배포 전 필수.** 컬럼이 없으면 Supabase insert 자체가 실패(unknown column)하고 `saveSurveyResponse`의
듀얼 라이트가 Blob 쪽으로만 저장된다 → 어드민(`/admin/surveys`, Supabase 기준)에서 응답이 안 보인다.
즉 "마이그레이션 안 해도 값만 누락"이 아니라 **응답 전체가 어드민에서 사라진다.** 반드시 먼저 실행할 것.
