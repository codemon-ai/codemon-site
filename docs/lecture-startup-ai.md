## M2. 비개발자도 앱을 만든다 — 바이브 코딩의 시대 (20분)

> 발표 목표: 바이브 코딩이 스타트업에 의미하는 바를 전달하고, 대표가 알아야 할 AI 범위를 정리

### 바이브 코딩이란?

**바이브 코딩(Vibe Coding)**: 자연어로 원하는 것을 설명하면 AI가 전부 구현하는 것

"코드를 쓰지 않고, 원하는 것을 말로 설명해서 앱을 만듭니다. 바이브(감)만 전달하면 AI가 코딩합니다."

### 시장 데이터

| 숫자 | 의미 |
|------|------|
| **$47억** | 2026년 바이브 코딩 시장 규모 |
| **$123억** | 2027년 전망 (CAGR 38%) |
| **63%** | 바이브 코딩 사용자 중 비개발자 비율 |
| **$100M ARR** | Lovable이 8개월 만에 달성 (역대 최고 속도 SaaS 성장) |

### 스타트업에 의미하는 것

1. **프로토타이핑 비용 → 0**: 아이디어를 검증하는 데 개발자 채용이 필요 없어짐
2. **가치 이동**: 클라이언트가 "이런 앱 만들어줘" 대신 직접 만들기 시작 → "만드는 것"의 가치가 하락하고 "잘 만드는 것 + 운영하는 것"의 가치가 상승
3. **인턴이 프로토타입을**: 코딩 경험 없는 팀원도 MVP를 만들어 올 수 있는 시대

"대표님이 새벽에 떠오른 아이디어를, 아침에 출근해서 AI한테 말하면 점심 전에 프로토타입이 나옵니다. 예전에는 개발자 채용부터 시작해야 했습니다."

### 대표가 알아야 할 AI vs 몰라도 되는 AI

| 알아야 할 것 | 몰라도 되는 것 |
|-------------|--------------|
| AI가 무엇을 할 수 있는지 | 모델 구조 (트랜스포머, 어텐션) |
| 비용은 얼마인지 | 파라미터 수 (70B, 405B) |
| 어디에 쓰면 효과적인지 | 토큰 계산법 |
| 어떤 AI를 써야 하는지 | 파인튜닝 방법론 |
| 품질을 어떻게 관리하는지 | 벤치마크 점수 (MMLU, HumanEval) |

**AI의 3단계 진화 — 비개발자 언어로**:

| 단계 | 비유 | 할 수 있는 것 |
|------|------|-------------|
| AI 챗봇 | 똑똑한 인턴 | 질문에 답변, 문서 요약, 아이디어 브레인스토밍 |
| AI 코딩 도구 | 실행력 있는 팀원 | 코드 작성, 파일 수정, 테스트 실행, 배포 |
| AI 팀 | 부서 하나 | 여러 AI가 역할 분담해서 프로젝트를 진행 |

## M3. AI 팀원의 작동 원리 — 연결, 역할, 매뉴얼 (20분)

> 발표 목표: MCP, 에이전트, CLAUDE.md, 스킬을 비개발자 비유로 설명

### MCP = "AI의 USB-C 포트"

**MCP (Model Context Protocol)**: AI 에이전트가 외부 도구/서비스에 접근하는 표준 프로토콜

> Anthropic이 "AI 앱의 USB-C 포트"라고 설명

"지금까지 AI는 '대화만 하는 직원'이었습니다. MCP를 연결하면 '회사 시스템에 로그인해서 직접 일하는 직원'이 됩니다."

**현황**:
- **1,000개+** 커뮤니티 서버
- OpenAI, Google, Microsoft 모두 채택 → 사실상 유일 표준
- OpenAI가 Assistants API를 2026년 중반 종료 예정 → MCP로 통합

### 구체 예시 3개

| 연결 | AI가 하는 일 | 스타트업 활용 |
|------|-------------|-------------|
| MCP + **Slack** | 메시지를 읽고, 요약하고, 답장 | 고객 문의 자동 응대, 팀 커뮤니케이션 요약 |
| MCP + **Google Calendar** | 일정 확인하고 회의를 잡음 | 미팅 스케줄링, 일정 충돌 체크 |
| MCP + **GitHub** | 코드를 읽고, PR을 만들고, 리뷰 | 코드 리뷰 자동화, 이슈 관리 |

### 에이전트 = 역할이 있는 AI 팀원

**서브에이전트**: Claude Code 안에서 특정 작업을 전담하는 격리된 AI 인스턴스

```
대표 (지시)
├── AI 리서처: "경쟁사 분석해줘"
├── AI 에디터: "블로그 글 써줘"
└── AI 개발자: "랜딩페이지 만들어줘"
```

**Agent Teams**: 서브에이전트의 진화형. AI끼리 **서로 직접 소통** 가능

```
[서브에이전트]                  [Agent Teams]
메인 ←→ A                    Lead ←→ A ←→ B
메인 ←→ B                              ↕
(A와 B는 직접 소통 불가)             C
                            (모두 직접 소통 가능)
```

"팀장이 매번 중계하지 않아도, 팀원끼리 직접 소통하면서 일합니다."

### CLAUDE.md = AI 온보딩 문서

새 팀원이 입사하면 온보딩 문서를 줍니다. AI 에이전트도 마찬가지.

**CLAUDE.md**: 프로젝트에 들어올 때 AI가 가장 먼저 읽는 파일

```markdown
# CLAUDE.md (예시)

## 프로젝트 개요
- 우리 회사: ○○ 스타트업
- 제품: B2B SaaS

## 업무 규칙
- 고객 데이터는 절대 외부로 보내지 않는다
- 배포 전 반드시 테스트 통과
- 커밋 메시지는 한국어로
```

"이 문서 하나만 잘 쓰면, AI가 우리 회사의 규칙을 이해하고 따릅니다."

### 스킬 = AI 업무 매뉴얼

**스킬(Skills)**: AI에게 가르쳐둔 업무 매뉴얼

사람으로 치면:
- 신입한테 "회의록은 이 양식으로 써" → 다음부터 알아서 함
- 인턴한테 "메일 요약은 이렇게 해" → 매번 안 알려줘도 됨

AI 스킬도 동일:
1. **자동으로 인식** — "회의록 정리해줘" 하면 스킬이 알아서 발동
2. **일관된 결과** — 매번 양식 설명 안 해도 됨
3. **재사용 가능** — 모든 프로젝트에서 같은 스킬을 사용

**비개발자도 만들 수 있음**: Claude Desktop에서 자연어로 3분이면 작성. 코딩 필요 없음.

### 🎮 데모 2: MCP 연결

"지금 Claude에 슬랙을 연결하겠습니다. 연결 후 '오늘 마케팅 채널에서 무슨 이야기가 나왔어?'라고 물어보겠습니다."

- 입력: Slack MCP 연결 + 자연어 질문
- 출력: 채널 메시지 요약
- 포인트: "회사 도구를 AI에 연결하면 이런 일이 가능합니다"

## M4. 5명이 50명을 이기는 구조 ★ (20분)

> 이 강의의 핵심 모듈. 실제 운영 중인 AI 팀 구조와 자동화 사례를 공개

### 실제 AI 팀 7명 소개

| AI 에이전트 | 역할 | 하는 일 | 월 비용 |
|------------|------|--------|---------|
| **로디** | PM & 퍼블리셔 | 기획, 블로그 발행, 이미지 생성, SNS 포스팅 | ~$0.5 |
| **누비** | QA & Dev | 코드 리뷰, 테스트, 디버깅, 외주 대응 | ~$0.5 |
| **나래** | 리서처 | 시장 조사, 경쟁사 분석, 트렌드 리포트 | ~$0.5 |
| **베어** | 디자이너 | UI/UX, 컴포넌트 설계, 프론트엔드 | ~$0.3 |
| **뉴비** | 주니어 개발자 | 반복 코딩, 컴포넌트 생성 | ~$0.3 |
| **옥토** | DevOps | 서버, API, 인프라, 배포 자동화 | ~$0.3 |
| **아울** | 코드 리뷰어 | PR 리뷰, 코드 품질 관리, 데이터 처리 | ~$0.3 |

> 이 팀의 월 운영 비용: **약 $3** (AI API 사용료)

"사람 7명을 채용하면 월 수천만원입니다. AI 7명은 월 4,000원입니다. 물론 AI가 사람을 완전히 대체하는 건 아닙니다. 하지만 1명이 7명의 AI와 함께 일하면, 혼자서는 절대 불가능한 일들이 가능해집니다."

### 자동화 파이프라인: 진화 과정

**블로그 글 하나를 만드는 파이프라인**:

```
[v1 — 초기] 36분
메모 작성 → 사람이 리서치 → 사람이 초안 작성 → AI가 이미지 → 수동 배포

[v2 — 중기] 13분
메모 작성 → 나래(리서치) 5분 → 로디(작성) 2분 → 누비(검수) 5분 → 자동 배포

[v3 — 현재] 9분
메모 작성 → 파이프라인 자동 트리거 → 리서치/작성/검수/배포 완전 자동
```

**핵심**: v1(36분) → v2(13분) → v3(9분), **75% 시간 단축**

"처음부터 완벽한 자동화를 만든 게 아닙니다. 단계적으로 발전시킨 겁니다. 스타트업도 이렇게 시작하면 됩니다."

### 2봇 협업에서 시작하는 현실적 접근

모든 스타트업이 7개 에이전트부터 시작할 필요 없음. **2대부터 시작**.

codemon의 시작: 맥미니 1대(로디) + 맥북 1대(누비)

**협업 구조**:
- iCloud 공유 폴더로 비동기 통신
- GitHub으로 스킬 공유
- 역할 비대칭: 로디=상주 서버 관리자, 누비=이동형 보조

**핵심 설계 원칙**:

| 원칙 | 이유 |
|------|------|
| 비동기 > 실시간 | 상시 연결 불필요한 환경에서 과잉 설계 방지 |
| 파일 > 프로토콜 | 디버깅 쉽고, 사람도 읽을 수 있고, 도구 의존성 없음 |
| 역할 분리 | 대칭 구조는 충돌을 유발. 비대칭이 자연스럽다 |

"화려한 인프라 없이도, 맥 2대면 AI 팀을 시작할 수 있습니다."

### 🎮 데모 3: 블로그 자동화

"제가 지금 텔레그램에서 한 줄 메시지를 보내겠습니다. AI 팀 3명이 자동으로 움직여서 블로그 글을 만듭니다."

- 입력: 텔레그램 한 줄 메시지
- 과정: 나래(리서치) → 로디(작성) → 누비(검수) → 자동 배포
- 소요: 9~13분
- 포인트: "메시지 하나로 리서치부터 배포까지 전 과정이 자동화됩니다"

### 🎮 데모 4: SNS 자동 포스팅

"블로그 글이 발행되면 자동으로 SNS 포스팅용 이미지와 텍스트가 만들어집니다."

- 입력: 발행된 블로그 글
- 과정: 핵심 추출 → 이미지 생성 → 플랫폼별 텍스트 작성 → 포스팅
- 포인트: "콘텐츠 하나를 만들면 유통까지 자동으로. 마케터 없이도 됩니다."

## M5. 품질 — 틀릴 수 있다, 그래서 시스템이 필요하다 (20분)

> 발표 목표: AI 품질 관리의 현실적 접근법과 자동 가드레일 시스템 소개

### 대전제: "AI는 틀린다. 문제는 어떻게 잡느냐."

"AI가 절대 틀리지 않는다고 말하면 거짓말입니다. 사람도 틀립니다. 중요한 건 '틀린 걸 어떻게 잡느냐'입니다. 사람 팀에는 코드 리뷰, QA, 매니저 검토가 있습니다. AI 팀에도 같은 구조를 만들면 됩니다."

### 이중 검증: AI 생성 → AI 검증 → 사람 승인

```
[AI 생성]                   [AI 검증]                  [사람 승인]
로디가 블로그 글 작성  →  누비가 팩트체크 + 검수  →  대표가 최종 확인
                            ↓
                     자동 테스트 실행
                     보안 점검 실행
                     스타일 가이드 확인
```

**핵심**: "AI 하나가 만들고, 다른 AI가 검증합니다. 이중 검증 후에 사람이 최종 승인합니다. 사람이 처음부터 만드는 것보다 빠르면서도 품질을 유지합니다."

### Hooks: 자동 가드레일

에이전트가 자율적으로 일할 때 필요한 자동 안전장치.

| 훅 | 시점 | 스타트업 활용 예시 |
|----|------|-----------------|
| **PreToolUse** | 도구 실행 전 | 프로덕션 DB 직접 수정 차단, 민감 파일 수정 경고 |
| **PostToolUse** | 도구 실행 후 | 코드 수정 시 자동 포맷팅 + 린트 + 테스트 |
| **SessionStart** | 세션 시작 | 프로젝트 컨텍스트 자동 로드 |
| **Stop** | 작업 완료 | 슬랙/텔레그램으로 결과 알림 |

**실제 예시**: "개발 에이전트가 코드를 수정하면 → 자동으로 포맷팅이 돌고 → 테스트가 실행되고 → 실패하면 에이전트가 다시 수정 → 성공하면 슬랙에 알림"

"사람에게 규칙을 반복해서 말해야 합니다. AI에게는 한 번만 설정하면 100% 지킵니다. 피곤하지도 않고, 까먹지도 않습니다."

### 보안: 최소 권한 원칙 + 민감 정보 제한

| 원칙 | 구현 방법 |
|------|----------|
| **최소 권한** | AI에게 필요한 권한만 부여 (읽기만, 특정 폴더만) |
| **민감 정보 제한** | API 키, 비밀번호 등 AI가 접근하지 못하게 설정 |
| **데이터 미학습** | Claude API/Pro 플랜에서 사용자 데이터로 모델 학습하지 않음 |
| **자동 보안 점검** | 코드 수정 시 보안 취약점 자동 탐지 |

### 🎮 데모 5: 바이브 코딩

"코드 한 줄 없이 랜딩페이지를 만들겠습니다. AI에게 '우리 스타트업 랜딩페이지 만들어줘'라고 말합니다."

- 입력: 자연어 설명
- 출력: 동작하는 랜딩페이지
- 포인트: "만드는 건 AI가 하고, 판단은 사람이 합니다"

### 🎮 데모 6: 자동 품질 점검

"AI가 만든 사이트를 다른 AI가 테스트합니다. 버그를 찾고, 리포트를 작성하고, 수정까지 합니다."

- 입력: 방금 만든 랜딩페이지
- 과정: QA 에이전트가 페이지 접속 → 기능 테스트 → 버그 리포트 작성
- 포인트: "만든 AI와 검증하는 AI가 다릅니다. 셀프 검증이 아닙니다."

## M6. 비용 — 솔직한 숫자 (20분)

> 발표 목표: 비용에 대한 현실적 이해와 ROI 판단 기준 제공

### 모델별 가격표

### 규모별 비용 시뮬레이션

| 단계 | 구성 | 월 비용 | 할 수 있는 것 |

### 비용 최적화 3가지

1. **모델 분리**: 복잡한 판단은 Opus, 일상 코딩은 Sonnet, 단순 작업은 Haiku
2. **프롬프트 캐싱**: 반복 요청 최대 90% 절감
3. **Agent Teams 선별 사용**: 일반 세션 대비 ~7배 토큰 → 정말 필요한 작업에만

### ROI: 신입 인건비의 1/10 ~ 1/20

| 항목 | 신입 개발자 | AI 팀 (풀 파이프라인) |
|------|-----------|---------------------|
| 월 비용 | 300~500만원 | 30~60만원 ($200~400) |
| 가용 시간 | 8시간/일, 주 5일 | 24시간/일, 주 7일 |
| 온보딩 | 1~3개월 | 즉시 (CLAUDE.md) |
| 멀티태스킹 | 1~2개 | 병렬 무제한 |
| 퇴사 리스크 | 있음 | 없음 |
| 창의적 판단 | **사람만 가능** | 불가 |

"AI가 사람을 대체한다는 뜻이 아닙니다. 사람 1명 + AI 7명이, 사람 1명만일 때보다 훨씬 많은 일을 한다는 뜻입니다."

### 직접 고용 vs AI 팀 vs 외주 비교표

| 기준 | 직접 고용 | AI 팀 | 외주 |
|------|----------|-------|------|
| 초기 비용 | 높음 (채용) | 낮음 ($20~) | 중간 (프로젝트별) |
| 월 고정비 | 높음 (급여) | 낮음 ($200~400) | 없음 (건당) |
| 즉시 투입 | 1~3개월 | 즉시 | 1~2주 |
| 품질 통제 | 높음 | 중간 (시스템 필요) | 낮음 |
| 확장성 | 느림 (추가 채용) | 즉시 (에이전트 추가) | 느림 (업체 변경) |
| 기술 축적 | 높음 | 중간 (CLAUDE.md에 축적) | 낮음 |

"정답은 없습니다. 핵심 역량은 사람, 반복 업무는 AI, 전문 일회성은 외주. 이 조합이 스타트업에 가장 효율적입니다."

## Q&A (25분)

### 예상 질문과 답변

**Q: "코드 모르는 사람도 할 수 있나요?"**
A: Lv.1은 누구나 가능합니다. Claude.ai에서 대화하고, Cowork에서 스킬을 만드는 건 코딩 경험 없이도 됩니다. 코딩 에이전트 활용은 개발자의 도움이 필요합니다.

**Q: "보안은 괜찮나요? 우리 코드가 학습에 쓰이나요?"**
A: Claude는 API/Pro 플랜에서 사용자 데이터로 모델을 학습하지 않습니다. 엔터프라이즈 플랜은 추가 보안 보장이 있습니다.

**Q: "비용이 계속 늘어나지 않나요?"**
A: 모델 분리(Opus/Sonnet/Haiku)와 프롬프트 캐싱으로 최적화하면 월 $200~400 수준으로 유지 가능합니다. ROI는 보통 첫 달부터 체감됩니다.

**Q: "우리 스타트업 규모(3~10명)에도 효과가 있나요?"**
A: 오히려 소규모 스타트업에 더 효과적입니다. 대기업은 도입 프로세스가 복잡하지만, 스타트업은 오늘부터 시작할 수 있습니다.

**Q: "어떤 업무부터 자동화하면 좋나요?"**
A: 반복적이고, 구조화되어 있고, 실패해도 치명적이지 않은 업무부터. 추천: 경쟁사 리서치, 콘텐츠 초안, 주간 보고서 자동 생성.

**Q: "AI가 만든 코드를 믿을 수 있나요?"**
A: 사람 코드도 리뷰합니다. AI 코드도 리뷰하면 됩니다. 이중 검증 + 자동 테스트로 품질을 보장하는 시스템을 만드는 게 핵심입니다.
