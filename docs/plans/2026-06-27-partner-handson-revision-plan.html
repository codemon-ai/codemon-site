파트너 핸즈온 강의 2종 수정 계획

작성: 2026-06-27 · 대상: lecture-claude-build, lecture-trading-bot · 상태: 완료

목표는 강의 내용을 최신 사실과 실제 수업 운영 흐름에 맞게 다듬는 것이다. 개요 페이지는 대상·산출물·준비물을 명확히 하고, HTML 가이드는 수강생이 막히지 않도록 Core / Fallback / Optional 경로를 분리한다.

2 Claude Build 강의 개요와 가이드 수정 DONE 자료 링크, 준비물, Node 설명, LLM Wiki, Skill/MCP/Cowork, 배포 성공 기준이 정리된다.

공통 설계 원칙

개요와 가이드 역할 분리

개요 페이지는 누가, 왜, 무엇을 만드는지 빠르게 판단하게 한다. HTML 가이드는 클릭 경로, 명령어, 실패 시 대체 루트를 제공한다.

Core / Fallback / Optional

모든 수강생이 끝낼 최소 경로를 Core로 두고, 계정·유료 플랜·설치 실패 시 Fallback, 시간이 남을 때 Optional을 둔다.

정책·가격 단정 완화

요금제, UI 경로, 무료 한도는 바뀔 수 있으므로 "수업일 기준 재확인" 문구와 공식 링크를 함께 둔다.

수업 완주율 우선

멋진 확장보다 수강생 전원이 동작 결과를 확인하는 것을 우선한다. 복잡한 기능은 선택 과제로 분리한다.

Claude Build 수정 범위

  • 개요 상단: "자료는 추후 추가" 문구를 제거하고 전체 가이드 링크를 명확히 유지한다.
  • 파트너 목록: /partner 목록에 Claude Build 전체 자료 링크를 추가한다.
  • 준비물: 필수 / 권장 / 선택을 분리한다. Claude 유료 플랜, GitHub/Vercel/Supabase, Node.js, Antigravity의 역할을 다시 쓴다.
  • Node.js: Desktop Extensions에는 별도 Node 설치가 필요 없을 수 있고, Claude Code npm 설치·로컬 사이트 빌드에는 필요하다고 분리한다.
  • LLM Wiki: RAG 대비 설명은 유지하고, 좋은 노트 구조와 Ingest / Query / Lint 사용 시점을 더 선명하게 한다.
  • Skill/MCP/Cowork: 용어 순서를 "Skill = 방법, MCP = 통로, Cowork = 맡기는 화면"으로 정리한다.
  • 사이트 배포: 성공 기준을 최소 / 표준 / 확장으로 나눠 4시간 운영 가능성을 높인다.

Trading Bot 수정 범위

  • 상단 경고: 실거래 전환·수익 전략·투자 조언이 아님을 더 구체적으로 반복한다.
  • 기본 경로: TradingView를 Optional로 낮추고, Core는 curl 시뮬레이션으로 잡는다.
  • TradingView 플랜: 현재 가격표 기준으로 Essential 이상으로 보이지만 변동 가능성을 명시하고 수업일 재확인을 넣는다.
  • ngrok: "재시작마다 무조건 변경" 대신 "터미널에 표시되는 forwarding URL 확인, 바뀌면 알림 URL 갱신"으로 완화한다.
  • 보안: 실습 코드는 passphrase 중심, 보안 강화 박스에 IP allowlist, 환경변수, rate limit, paper guard를 둔다.
  • 대시보드: Streamlit은 시간이 부족할 수 있으므로 Optional 또는 확장 성공 기준으로 분리한다.
  • 정직 포인트: 만들지 않는 것과 만드는 것을 분리해 "돈 버는 봇" 오해를 줄인다.
