신호가 오면 자동으로 주문하는나만의 투자 봇

TradingView 신호 → 내 웹훅 서버 → Alpaca 페이퍼 자동 주문 → 텔레그램 알림 + 대시보드. 코드 한 줄씩 따라 만들고, 실제 돈 없이 모의계좌에서 동작까지 확인하는 2시간입니다.

⚠️ 먼저 읽어주세요 — 이 수업은 모의(페이퍼) 투자 전용입니다

이 과정의 모든 코드는 Alpaca 페이퍼 트레이딩(가짜 돈)으로 동작합니다(paper=True). 투자 조언이 아니며, 교육·프로토타입 목적입니다. 실거래 전환은 본인 책임이며 손실 위험이 크고, 자동매매·레버리지는 특히 위험합니다. API 키는 절대 공유·하드코딩하지 마세요.

전부 공식 문서로 확인한 현행 코드입니다. (주의: 유명 튜토리얼 레포는 구버전 alpaca-trade-api — 우리는 현행 alpaca-py로.)

① 첫 주문 — Alpaca 페이퍼 alpaca-py
# pip install alpaca-py
from alpaca.trading.client import TradingClient
from alpaca.trading.requests import MarketOrderRequest
from alpaca.trading.enums import OrderSide, TimeInForce

client = TradingClient("KEY", "SECRET", paper=True)   # 모의계좌

order = MarketOrderRequest(symbol="AAPL", qty=1,
        side=OrderSide.BUY, time_in_force=TimeInForce.DAY)
client.submit_order(order_data=order)

print(client.get_account().portfolio_value)   # 계좌
print(client.get_all_positions())              # 포지션
② 웹훅 서버 — 신호 받으면 주문 Flask
# pip install flask alpaca-py
from flask import Flask, request, abort
from alpaca.trading.client import TradingClient
from alpaca.trading.requests import MarketOrderRequest
from alpaca.trading.enums import OrderSide, TimeInForce

PASS = "my-long-random-token"   # TV 웹훅엔 인증이 없어 직접 방어
app = Flask(__name__)
client = TradingClient("KEY", "SECRET", paper=True)

@app.route("/webhook", methods=["POST"])
def webhook():
    d = request.get_json(force=True)
    if d.get("passphrase") != PASS: abort(403)
    side = OrderSide.BUY if d["action"] == "buy" else OrderSide.SELL
    o = MarketOrderRequest(symbol=d["ticker"], qty=1,
        side=side, time_in_force=TimeInForce.GTC)
    client.submit_order(order_data=o)
    return {"status": "ok"}, 200

app.run(host="0.0.0.0", port=5000)
③ 공개 + 신호 쏘기 ngrok curl 시뮬

ngrok http 5000
# → https://xxxx.ngrok-free.app  (재시작하면 URL 바뀜!)

curl -X POST https://xxxx.ngrok-free.app/webhook \
  -H "Content-Type: application/json" \
  -d '{"passphrase":"my-long-random-token","action":"buy","ticker":"AAPL"}'

⑤ 대시보드 Streamlit
# pip install streamlit alpaca-py pandas  →  streamlit run app.py
import streamlit as st, pandas as pd
from alpaca.trading.client import TradingClient

c = TradingClient("KEY", "SECRET", paper=True)
acc, pos = c.get_account(), c.get_all_positions()

st.title("내 트레이딩 대시보드")
st.metric("포트폴리오 가치", f"${acc.portfolio_value}")
st.dataframe(pd.DataFrame([{"종목": p.symbol, "수량": p.qty,
  "평가손익": p.unrealized_pl} for p in pos]))

정직 포인트 · 리스크

과장하지 않고 사실대로

  • TradingView엔 '사용자용 체결 API'가 없습니다자동매매의 표준 경로는 알림 → 웹훅 → 브로커(Alpaca) API입니다. TradingView 공식 Broker API는 '브로커가 자사를 TradingView에 연동'하는 프레임워크지, 우리가 주문 내는 API가 아닙니다.
  • "TradingView MCP"는 전부 비공식입니다커뮤니티 제작(예: bidouilles/mcp-tradingview-server 등)이고 스크래핑 기반이라 ToS 리스크가 있습니다. 데이터 라이브러리(tvdatafeed·tradingview-ta)도 비공식이며 일부는 유지보수 종료. 교육/프로토타입용으로만.
  • 가장 유명한 튜토리얼은 구버전 SDK입니다hackingthemarkets 레포(★189)는 deprecated alpaca-trade-api를 씁니다. 패턴은 거기서 배우되 현행 alpaca-py로 작성하세요.
  • 웹훅은 누구나 POST할 수 있습니다TradingView 웹훅엔 서명 인증이 없습니다. 1차 방어로 passphrase 검증 + 발신 IP 4개 화이트리스트, 그리고 데모는 반드시 paper=True. 키는 환경변수로.
  • 백테스트가 좋다고 미래가 좋은 건 아닙니다과최적화(curve fitting)·생존편향을 조심하고, 항상 모의계좌에서 충분히 검증하세요. 이 수업의 목표는 '돈 버는 봇'이 아니라 '파이프라인을 직접 만들어보는 것'입니다.
