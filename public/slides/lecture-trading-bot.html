실습 가이드 2시간 · 입문 📄 페이퍼(모의) 전용

자동 투자 봇페이퍼 주문까지 직접 확인

curl 신호를 보내고, Flask 서버가 Alpaca 페이퍼 주문과 텔레그램 알림까지 처리하는 과정을 직접 확인합니다. 필요한 명령어·코드·확인 방법을 순서대로 모았습니다.

⚠️ 이 가이드는 모의(페이퍼) 투자 전용입니다

모든 코드는 Alpaca 페이퍼(가짜 돈)로 동작합니다(paper=True). 투자 조언·수익 전략·실거래 전환 수업이 아닙니다. 수업에서는 신호→웹훅→모의 주문 파이프라인만 다룹니다. API 키는 절대 공유·하드코딩하지 마세요(환경변수 사용).

계정 · 키 준비

STEP 1키 3종 + 환경 만들기20분

목표 · Alpaca 페이퍼 키, 텔레그램 봇 토큰·chat_id, 파이썬 패키지를 준비한다.
A. Alpaca 페이퍼 계정 & API 키
  1. app.alpaca.markets/signup 에서 가입
    전 세계 누구나 무료로 페이퍼 계정 생성 가능.
  2. 좌측 상단에서 "Paper" 계정으로 전환
    Live(실거래)가 아닌 Paper인지 반드시 확인.
  3. API Keys 패널 → 키 생성 → Key ID + Secret Key 복사
    Secret은 이때 한 번만 보입니다. 즉시 저장하세요. 분실 시 재발급하면 기존 키는 무효화됩니다.
Base URL은 https://paper-api.alpaca.markets (REST 경로는 /v2/...). 페이퍼 환경은 무료입니다.
B. 텔레그램 봇 만들기 BotFather
  1. 텔레그램에서 @BotFather 와 대화 시작
  2. /newbot 전송
  3. 봇 이름(name) 입력
    나중에 변경 가능.
  4. 사용자명(username) 입력
    5~32자, 라틴문자·숫자·언더스코어만, 반드시 bot으로 끝나야 함 (예: my_trade_bot).
  5. 토큰 받기
    예: 110201543:AAHdqTc... — 안전하게 보관.
  6. chat_id 얻기
    봇에게 아무 메시지나 보낸 뒤 https://api.telegram.org/bot<TOKEN>/getUpdates 를 브라우저로 열면 result[].message.chat.id 가 chat_id.
브라우저에서 https://api.telegram.org/bot<TOKEN>/getMe 를 열어 봇 정보가 나오면 토큰 정상.
C. 파이썬 패키지 설치
pip install alpaca-py flask requests streamlit
TradingView 웹훅을 실제로 쓰려면 웹훅을 지원하는 유료 플랜 + 2FA(2단계 인증)가 필요합니다. 현재 가격표 기준 Basic은 미지원, Essential 이상에서 지원되는 것으로 보이지만 수업일 가격표를 다시 확인합니다. 없으면 STEP 3의 curl 시뮬레이션으로 동일하게 실습합니다.
첫 주문

STEP 2Alpaca에 모의 주문 넣기20분

목표 · 내 파이썬 코드로 Alpaca 페이퍼 주문이 들어가는지 확인한다.
first_order.py alpaca-py
# pip install alpaca-py
from alpaca.trading.client import TradingClient
from alpaca.trading.requests import MarketOrderRequest
from alpaca.trading.enums import OrderSide, TimeInForce

client = TradingClient("YOUR_KEY", "YOUR_SECRET", paper=True)  # 모의계좌

order = MarketOrderRequest(
    symbol="AAPL", qty=1,
    side=OrderSide.BUY, time_in_force=TimeInForce.DAY,
)
client.submit_order(order_data=order)

print(client.get_account().portfolio_value)   # 계좌 가치
print(client.get_all_positions())              # 보유 포지션
실행: python first_order.py
Alpaca 웹 대시보드(app.alpaca.markets)의 Orders / Positions 페이지에 방금 주문이 보이면 성공.
속성 철자(portfolio_value, unrealized_pl 등)는 SDK 버전에 따라 다를 수 있습니다. 수업 전 실제 키로 한 번 실행해 vars(account)로 확정하세요.
웹훅 봇

STEP 3 ★신호가 오면 자동 주문35분

목표 · 외부에서 신호(JSON)를 받으면 자동으로 주문하는 웹훅 서버를 띄운다.
순서경로누가 하나
1curl로 신호 쏘기모두 같이
2TradingView 알림 JSON 연결유료 플랜 + 2FA 준비자
3IP allowlist·환경변수·rate limit 보강시간 남으면
① 웹훅 서버 — app.py Flask
# pip install flask alpaca-py
from flask import Flask, request, abort
from alpaca.trading.client import TradingClient
from alpaca.trading.requests import MarketOrderRequest
from alpaca.trading.enums import OrderSide, TimeInForce

PASS = "my-long-random-token"   # TV 웹훅엔 인증이 없어 직접 방어
app = Flask(__name__)
client = TradingClient("YOUR_KEY", "YOUR_SECRET", paper=True)

@app.route("/webhook", methods=["POST"])
def webhook():
    d = request.get_json(force=True)
    if d.get("passphrase") != PASS:
        abort(403)
    side = OrderSide.BUY if d["action"] == "buy" else OrderSide.SELL
    o = MarketOrderRequest(symbol=d["ticker"], qty=1,
        side=side, time_in_force=TimeInForce.GTC)
    client.submit_order(order_data=o)
    return {"status": "ok"}, 200

app.run(host="0.0.0.0", port=5000)
실행: python app.py → 로컬 5000 포트에서 대기.
② 인터넷에 공개 ngrok
ngrok http 5000
# → https://xxxx.ngrok-free.app  (이 URL + /webhook 이 내 봇 주소)
터미널에 표시되는 Forwarding URL을 그대로 사용합니다. URL이 바뀌면 TradingView 알림에 넣은 URL도 갱신해야 합니다.
③ 신호 쏘기 — (A) curl 시뮬 기본
curl -X POST https://xxxx.ngrok-free.app/webhook \
  -H "Content-Type: application/json" \
  -d '{"passphrase":"my-long-random-token","action":"buy","ticker":"AAPL"}'
④ 신호 쏘기 — (B) TradingView 알림 선택
  1. 차트에서 알림 생성
    단축키 Alt+A (Mac ⌥+A).
  2. 알림의 Notifications 영역 → "Webhook URL" 체크 → ngrok URL + /webhook 입력
  3. 메시지(JSON) 작성
    숫자는 따옴표 없이, 문자열은 따옴표로(valid JSON):
{ "passphrase": "my-long-random-token",
  "action": "{{strategy.order.action}}",
  "ticker": "{{ticker}}" }
TradingView 웹훅 요건: 포트 80/443(HTTPS)만, 3초 타임아웃, 2FA 활성 시에만 가능. 발신 IP 4개: 52.89.214.238 / 34.212.75.30 / 54.218.53.128 / 52.32.178.7.
curl(또는 알림) 발사 → Flask 콘솔에 요청 로그 + Alpaca Orders에 주문이 들어오면 성공.
보안: 수업용 코드는 이해를 위해 passphrase 검증 + paper=True만 넣습니다. 운영형 프로토타입은 발신 IP allowlist, 환경변수, rate limit, 주문 수량 제한을 추가해야 합니다.
텔레그램 알림

STEP 4체결되면 폰으로 알림15분

목표 · 주문이 들어가면 텔레그램으로 "방금 매수했어요" 알림을 받는다.
알림 함수 + 웹훅에 통합 Bot API
import requests
TOKEN, CHAT = "123:ABC", "987654321"

def notify(msg):
    requests.post(f"https://api.telegram.org/bot{TOKEN}/sendMessage",
                  data={"chat_id": CHAT, "text": msg})

# app.py 의 webhook() 안, submit_order 직후:
client.submit_order(order_data=o)
notify(f"✅ {d['action'].upper()} {d['ticker']} 1주 체결 (paper)")
curl로 신호 쏜 뒤 텔레그램에 "✅ BUY AAPL 1주 체결 (paper)" 메시지가 오면 성공. (텔레그램 Bot API는 전부 무료)

STEP 5내 포트폴리오 화면15분

목표 · 시간이 남으면 포트폴리오 가치·보유 포지션·손익을 한 화면에 본다.
가장 단순한 건 Alpaca 자체 페이퍼 대시보드(코드 0줄). 여기선 직접 만들기를 위해 Python 하나로 끝나는 Streamlit을 씁니다.
dashboard.py Streamlit
# pip install streamlit alpaca-py pandas  →  streamlit run dashboard.py
import streamlit as st, pandas as pd
from alpaca.trading.client import TradingClient

c = TradingClient("YOUR_KEY", "YOUR_SECRET", paper=True)
acc, pos = c.get_account(), c.get_all_positions()

st.title("내 트레이딩 대시보드")
st.metric("포트폴리오 가치", f"${acc.portfolio_value}")
st.metric("현금", f"${acc.cash}")
st.dataframe(pd.DataFrame([{
    "종목": p.symbol, "수량": p.qty,
    "평가금액": p.market_value, "평가손익": p.unrealized_pl
} for p in pos]))
실행: streamlit run dashboard.py → 브라우저 localhost:8501
위 속성명(portfolio_value·cash·market_value·unrealized_pl)은 실제 키로 한 번 실행해 철자를 확정하세요(SDK 버전 차이).

  • 다음으로 키울 것전략(Pine Script), 손절·익절(브래킷 주문), 주문 로깅·DB, 에러 처리·재시도, 포지션 사이징. 단, 실거래 전환은 신중히.
수업 범위

여기까지만 합니다

  • TradingView엔 '사용자용 체결 API'가 없습니다자동매매 표준 경로는 알림 → 웹훅 → 브로커(Alpaca) API입니다. TradingView 공식 Broker API는 '브로커가 자사를 TradingView에 연동'하는 프레임워크지, 우리가 주문 내는 API가 아닙니다.
  • "TradingView MCP"는 전부 비공식입니다커뮤니티 제작(예: bidouilles/mcp-tradingview-server)이고 스크래핑 기반이라 ToS 리스크가 있습니다. 데이터 라이브러리(tvdatafeed·tradingview-ta)도 비공식·일부 유지보수 종료. 교육/프로토타입용으로만.
  • 가장 유명한 튜토리얼은 구버전 SDK입니다hackingthemarkets 레포(★189)는 deprecated alpaca-trade-api를 씁니다. 패턴은 거기서 배우되 현행 alpaca-py로 작성하세요.
  • TradingView 플랜과 ngrok URL 정책은 바뀔 수 있습니다수업일 가격표와 ngrok 콘솔의 Forwarding URL을 기준으로 안내합니다.
